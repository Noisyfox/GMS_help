Debugging 调试

本章节函数只是关于使用调试功能呢的。

当做游戏是，它往往是非常重要的，你知道在特定的点在时间的游戏机制的某些细节，或者能够输入特定键值的一些功能来测试不同的方法和思路，为您的游 戏。为此游戏制作工作室为您提供了一些调试功能，将显示消息，并要求在调试模式下输入（或在一个Windows目标），但在编译时将被忽略，由于这一事 实，适用于iOS，Android或HTML5这些功能在一个封闭的循环中放了游戏的，直到输入解决。

NOTE : 虽然他们可能在某些平台上可以使用，但他们是不被用于调试以外的任何东西。

  1. debug_mode
  2. get_integer
  3. get_string
  4. show_error
  5. show_message
  6. show_question
  7. show_debug_message

除了那些专业的功能，来帮助你创建你的游戏时，GameMaker:Studio也有一个长长的清单可以显示您的游戏或游戏本身运行 时编译时的错误信息。这些消息指向特定的错误的问题，并可以用来精确定位的问题，在一个特定的对象或脚本，大大减少调试你的游戏的进程。但是，它可能并不 总是很明显的错误消息意味着什么，所以以下两个部分列出了所有可能的消息，并解释他们的意思是什么使你的生活更轻松：

  1. Compile Errors 编译错误
  2. Runner Errors 运行错误
