Runner Errors 运行错误

在这里，你可以看到虚拟机的运行可以抛出的可能的错误。

即使语法检查后在代码编辑器，然后让编译器检查你的代码，仍然有偶然东西可以去错。在大多数情况下，这将抛出一个VM（虚拟机）这样的错误： VM的误差通常比编译错误更严重，因为它意味着有严重错误的东西与你的 代码，无论是代码编辑器，也不是编译器已经能够探测到，因此你要注意这样的错误。当一个人时，你可以使用“复制”按钮，在弹出错误复制到剪贴板，然后你就 可以粘贴到一个文本文件（或其他地方），以供将来参考。从VM的运行可能发生的错误如下：

NOTE : 所有的运行错误将尝试和指向导致崩溃的你的对象或脚本。

  1. Conversions

    DoConv :: Execution Error

    这表示一个数据类型转换成另一种中的错误。

  2. Addition

    DoAdd :: Execution Error

    当使用加号（+）的表达时某些发生了错误。

  3. Multiplication

    DoMul :: Execution Error

    当使用乘号（*）的表达时某些发生了错误。

  4. Subtraction

    DoSub :: Execution Error

    当使用减号（-）的表达时某些发生了错误。

    DoSub :: Execution Engine - Cannot operate on string type

    您正在尝试减去了错误类型的变量（例如从一个真正的减去一个字符串）。

  5. Division

    DoDiv :: Execution Error

    当使用的分工（/或div）的表达时发生了错误。

    DoDiv :: Execution Engine - Cannot operate on string type

    你正试图分裂了错误类型的变量（例如一个真正的分割字符串）。

    DoDiv :: Divide by zero

    您正试图除以0。

  6. Remainders

    DoMod :: Execution Error

    使用模（MOD）的表达时发生了错误。

    DoMod :: Execution Engine - Cannot operate on string type

    您正在尝试使用模错误类型的变量（例如一个真正的MOD字符串）。

  7. Bitwise And

    DoAnd :: Execution Error

    使用位“与”（＆）的表达时发生了错误。

    DoAnd :: Execution Engine - Cannot operate on string type

    您正在尝试使用“和”错误类型的变量（例如试图以“和”一个真正的字符串）。

  8. Bitwise Or

    DoOr :: Execution Error

    使用按位“或”（|）的表达时发生了错误。

    DoOr :: Execution Engine - Cannot operate on string type

    您正在尝试使用“或”错误类型的变量（例如试图“或”一个真正的字符串）。

  9. Bitwise Xor

    DoXor :: Execution Error

    使用按位“异或”（^）的表达时发生了错误。

    DoXor :: Execution Engine - Cannot operate on string type

    您正在尝试使用“异或”错误类型的变量（例如试图“XOR”一个真正的字符串）。

  10. Bitshift Left

    DoShl :: Execution Error

    左右bitshifting（<<）的值时发生了错误。

    DoShl :: Execution Engine - Cannot operate on string type

    您正在尝试：左边的bitshift了错误类型的变量（例如试图的bitshift一个字符串）。

  11. Bitshift Right

    DoShr :: Execution Error

    bitshifting右（>>）的值时发生了错误。

    DoShr :: Execution Engine - Cannot operate on string type

    您正在尝试：正确的bitshift了错误类型的变量（例如试图的bitshift一个字符串）。

  12. Negative

    DoNeg :: Execution Error

    您正试图把一个变量的类型转负时，这种类型的不允许这样操作。

  13. Not

    DoNot :: Execution Error

    您正试图为“否”时，这种类型的变量类型不允许这样操作。

  14. Stack Push

    Push :: Execution Error - Variable Index out of range (var)

    变量被允许超出运行器有效范围。

    Push :: Execution Error - Variable Get (var)

    给定的变量还没有被定义，或者是未知的。

  15. Stack Pop

    Pop :: Execution Error - Variable Index out of range (var)

    变量被允许超出运行器有效范围。

    Pop :: Execution Error - Variable Get (var)

    给定的变量还没有被定义，或者是未知的。

  16. With

    Cannot use global in with statement

    已尝试使用一个全局变量（即不指向一个有效的实例ID）在“with”语句。

    Cannot use local in with statement

    您尝试使用本地，为“var”声明的变量（即不指向一个有效的实例ID）在“with”语句

  17. General

    DoCall :: Execution Engine type error

    在虚拟机中，这是一个不确定的错误。你应该提交一个bug报告应发生这种情况（见：如何做到这一点的详细信息的 The Help Menu）。

    18. Stack Overflow...

    堆 栈溢出发生时，太多的内存被用来调用堆栈上和你的游戏时，试图以使用更多的空间比是可调用堆栈（即是，当它试图访问内存调用堆栈的边界之外，这基本上是一 个缓冲区溢出），堆栈溢出说，导致在一个程序崩溃。重新启动您的计算机和GameMaker:Studio，如果错误仍然存在，请在触摸的支持和/或提交 一个bug（如上所述）。
