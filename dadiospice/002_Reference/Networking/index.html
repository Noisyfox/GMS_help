Networking 联网

这些函数提供给你在游戏中基本的客户机/服务器网络。

GameMaker:Studio网络功能都是基于套接字，这是一个在所有平台上的行业标准。有了套接字，你有能力来创建客户端和服务器使用简单的GML它允许你创建一个单人游戏使用的客户机/服务器模型（这将允许多人后很容易地添加）。

基本上套接字是一种对象，它可以发送、接收、连接和收听网络端口。这些函数处理只用TCP / IP连接，这是基于互联网（IPv4向更精确）。在本质上，这可以让你连接一个套接字另一个IP地址（192.168.1.11）上使用IP地址（比方说 192.168.1.10）到另一个套接字。一旦连接，这两个套接字可以发送数据来回使用network_send_ *函数（如 network_send_packet) 和专用的异步事件，即触发。这种异步的事件将产生一个 ds_map 收到的数据，以及其他细节，你可以找到完整的 Network Async Event 细节。

相反的程序读取和处理网络流量进入一台机器的每一个位，IP地址也处理端口，端口编号从0到65535。这允许每个程序只得到它有兴趣的数据包，而不是所有已发送的每个程序。这也节省了大量的CPU时间，因为它大量减少了数据处理，GameMaker:Studio中，您也可以“listen”端口。因此，服务器会创建一个套接字，然后将其绑定到一个特定的端口来监听。然后，它会得到通知的连接，断开，和来至套接字的数据传输。


数据“包”本身是由取自一个缓冲的二进制数据。这个缓冲器可以创建和操纵使用的 GameMaker:Studio Buffer functions。



NOTE: 这些函数目前尚未提供HTML5的目标模块。

NOTE : 如果您使用的是免费版本的 GameMaker:Studio 你被限制为1的客户机/服务器连接。欲了解更多详细信息，请参阅 here。

存在以下函数供您使用网络：

  1. network_create_server
  2. network_create_socket
  3. network_connect
  4. network_connect_raw
  5. network_resolve
  6. network_set_timeout
  7. network_send_broadcast
  8. network_send_packet
  9. network_send_raw
  10. network_send_udp
  11. network_destroy
