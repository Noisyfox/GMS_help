Obsolete Transition Functions 淘汰过渡函数

这些函数是处理房间之间的过渡效果。

Function List :

FunctionDescription
transition_kind设置进入下个房间过渡显示效果，你有很多内置设置使用。
transition_steps表示在过渡的步的数目。越多的过渡需要的时间越长。默认值是80。
transition_define实 际上，你可以创建自己的过渡。为此，你必须定义一个脚本（可能的扩展包），以做过渡。有了这个功能，你就可以添加到系统的过渡。一种是过渡的索引（无论是 一个新的或现有的转换）。名称是脚本的名称。请注意，该脚本的名称是一个字符串！因此，必须有引号。请注意，这是真正先进的东西。脚本必须采取五个参数： 一个表面与所述图像的先前的房间，隔壁房间的图像的表面的宽度，高度的表面，和过渡的馏分（0和1之间的表面与）。然后，它必须使用两个表面绘制图像。
transition_exists返回过渡类型是否存在。

为什么删除这些

由于这一事实，他们或多或少的“接管”控制设备，这些函数已被删除。他们也不相容GMS处理图形的方式。
