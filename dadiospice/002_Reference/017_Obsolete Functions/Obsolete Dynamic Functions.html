Obsolete Dynamic Functions 淘汰动态函数

这些函数是处理在运行过程中的一般资源。

Object Function List :

FunctionDescription
object_add添加一个新的对象。返回对象的索引。你可以现在使用这个索引，用上面的函数设置对象的属性，然后可以使用索引创建对象的实例。
object_delete删除指定索引的对象。确保在任何房间中都没有对象的实例。
object_event_add我们必须为对象定义事件来赋予对象行为。你可以为事件添加动作代码。你需要指定对象，事件类型，事件数（使用的常量在之前的 event_perform() 函数里指定）。最后准备代码编译。你可以给每个事件添加多个代码动作。
object_event_clear你可以使用这个函数来清除指定事件的所有动作。

Variable Function List :

FunctionDescription
variable_global_exists返 回值为名为参数 name （字符串）的全局变量是否存在。                                                                                  
variable_local_exists返回值为当前实例的名为参数 name （字符串）的局部变量是否存在。
variable_global_get返回值为给出参数 name （字符串）的全局变量的值。
variable_global_array_get返回值为给出参数 name （字符串）的全局数组的参数 ind 的索引值。
variable_global_array2_get返回值为给出参数 name （字符串）的全局二维数组变量的参数 ind1 ， ind2 的索引值。
variable_local_get返回值为给出参数 name （字符串）的局部变量的值。
variable_local_array_get返回值为给出参数 name （字符串）的局部数组的参数 ind 的索引值。
variable_local_array2_get返回值为给出参数 name （字符串）的局部二维数组变量的参数 ind1 ， ind2 的索引值。
variable_global_set设置给出参数 name （字符串）的全局变量的值。
variable_global_array_set设置给出参数 name （字符串）的全局数组的参数 ind 的索引值。
variable_global_array2_set设置给出参数 name （字符串）的全局二维数组变量的参数 ind1 ， ind2 的索引值。
variable_local_set设置给出参数 name （字符串）的局部变量的值。
variable_local_array_set设置给出参数 name （字符串）的局部数组的参数 ind 的索引值。
variable_local_array2_set设置给出参数 name （字符串）的局部二维数组变量的参数 ind1 ， ind2 的索引值。

Resource Function List :

FunctionDescription
sprite_add_sprite增 加精灵存储文件页面到设定的精灵资源。这个文件必须是.gmspr后缀的从GM的精灵中保存出来的。                                   
background_add_background增加背景存储文件页面到设定的背景资源。这个文件必须是.gmbck后缀的从GM的背景中保存出来的。

Miscellaneous Function List :

FunctionDescription
room_set_code设置指定索引的房间的初始化代码字符串。
script_get_text返回值为指定索引号的脚本的文本。
execute_string执行字符串的代码。
execute_file执行文本文件的代码。
execute_program(程式名称,参数,等待)执行程式，可以用参数，等待决定是否等待程式执 行完毕。
execute_shell(程式名称,参数)在壳执行程式或者文件。
timeline_moment_add在步的瞬间给时间轴添加一个代码动作。参数 codestr 包含动作的代码。如果这步不存在它就被创建。所以你可以在同一时刻添加多个代码动作。

为什么淘汰这些

这些函数不能被使用在GMS的底层运行下。，GMS使用了一般设备的特殊打包，它们不可能使的对象和代码“飞速”。
