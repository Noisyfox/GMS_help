Obsolete Miscellaneous Functions 淘汰杂项函数

这些函数跟之前的GameMaker有关。

Variable Function List :

FunctionDescription
secure_mode返回游戏是否在安全模式运行。
set_program_priority设置程序的优先级。可以指定 -3 到 +3 之间的值。值为 -3 时意味着程序只会在如下情况运行，电脑没有其他程序要处理时，或是所有其他的程序都空闲时。值为 -2 或 -1 时在正常线下，其他程序会有优先权。值为 0 时是正常的值。值为 +1 或 +2 为较高的优先级，可能会导致较高的速度和较平滑的游戏流程。但其他程序会得到很少的处理时间。值为 +3 时为实时模式。在实时模式中基本上所有的时间都分配到游戏当中去了。这样会导致电脑正在运行的其他应用程序发生严重的问题。当然键盘事件不再被电脑记录下来。所以只有当你想要所有的处理时间时才使用这个函数。同样要注意最好使用前仔细的检查一下并且运行前保存游戏 。
set_application_title设置应用程序的标题对给定的字符串。例如显示在任务栏里的标题。
export_include_file导出文件的名字fname。这必须是一个字符串变量，所以不要忘了引号
export_include_file_location导出所包含的文件的名字fname中给定的位置。位置必须包含路径和文件名。
discard_include_file丢弃的名字fname中所包含的文件，释放内存使用。这必须是一个字符串变量，所以不要忘了引号。
gamemaker_pro指示是否已经创建游戏在标准版上。
gamemaker_registered等同于 gamemaker_pro.
gamemaker_version游戏制作的版本。这是一个整数。对于版本8.0，这可能是在800和809之间的任何东西。对于8.1版本，这将是810和819之间的事情，所以从来没有检查一个具体的版本，但更好的检查范围内。 800前，这个变量是不可用的版本。
error_last用字符串显示最后出错的信息。
error_occurred显示是否有错误发生。
sleep睡眠（ numb ）毫秒
io_handle操作用户 io，更新键盘及鼠标状态。
keyboard_wait等待，直到用户按下键盘上的某个键
mouse_wait等待，直到用户按下某个鼠标键。
disk_size返回驱动盘大小按字节。
disk_free返回驱动盘剩余空间大小按字节。
date_current_date返回日期对应于当前的时间。
date_current_time返回时间对应于当前的时间。
date_create_date创建日期时间值对应到指定的日期。
date_create_time创建日期时间值对应到指定的时间。
date_valid_date返回指定的日期是否是有效的。
date_valid_time返回指定的时间是否是有效的。

为什么淘汰这些

这些函数不能被使用在GMS的底层运行下。这些都是只能在window中使用功能，不再适用于任何其他设备。
