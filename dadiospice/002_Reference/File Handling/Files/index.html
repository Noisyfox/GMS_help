Text Files 文本文件

本章节的说明处理文本文件的函数。

从您的游戏存储大量的数据到外部，文本文件是一种有用的方法。ini文件相反，你可以有多达32个文本文件一次打开，并且您可以读取和写入大量的数据到它们。但是，它是由你来决定按如何的数据结构，没有类似 *.ini files 设置“节”或“关键”的结构。

请注意，在游戏本地化上，它是非常重要的，允许从外部文件读取字母。这意味着，你应该创建UTF8格式的txt文件，然后将其添加到GameMaker:Studio中包含的文件，因此，它扩展到运行游戏，用于替代使用默认的ANSI格式的txt文件，如果没有文件就用GameMaker:Studio以前提供的文件的函数创建。通过这种方式，您可以正常读取和写入到它所在的地方语言和非罗马字母。 下面的函数存在，处理文件：

存在以下函数处理文本文件：

  1. file_text_open_read
  2. file_text_open_write
  3. file_text_open_append
  4. file_text_read_real
  5. file_text_read_string
  6. file_text_readln
  7. file_text_write_real
  8. file_text_write_string
  9. file_text_writeln
  10. file_text_eoln
  11. file_text_eof
  12. file_text_close


WARNING! 这些函数可能无法按预期在 GameMaker:Studio 沙箱中工作！更多信息请参见文件系统限制  File System Limits 。

NOTE : 这些函数被限制在标准版 Standard Version 的 GameMaker:Studio 及其以上。
