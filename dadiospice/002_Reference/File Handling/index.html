File Handling 文件处理

本章节关于文件处理的不同函数。

在某些时候，您的游戏需要保存某些额外信息并且检索这些信息。比如，您会喜欢存储登录信息，或高分榜，或玩家等级。无论什么原因，您需要知道怎样使用不同的文件函数在GameMaker:Studio中。

非常重要的认识是 GameMaker:Studio是一个沙盒，意味着有一些限制和轻微并发症当使用文件系统函数时，值得一提的是在不同的平台上文件函数工作略有不同。欲知更多信息，请见后面章节：

  1. File System Limits 文件系统限制

NOTE : 这些函数仅限 Standard Version 及以上版本。

文件系统

目录

显然，有时候你需要引用的文件存储和GameMaker:Studio有可能在不同的目录，你需要一些特殊函数：

  1. directory_exists
  2. directory_create
  3. temp_directory
  4. working_directory
  5. program_directory

编码

在处理外部文件时，总有可能有用户打开和修改这些文件所包含的信息，并因此改变你的游戏的可能性。这可能会导致打破游戏规则或欺诈分数线（例如），所以GameMaker:Studio 为您提供了一些基本的编码函数以及字符串和文件进行MD5检查函数，以确保他们一直保持着在使用前的完整性。还有两个函数提供的编码和解码JSON（JavaScript对象符号）格式字符串。

  1. base64_encode
  2. base64_decode
  3. json_encode
  4. json_decode
  5. md5_string_utf8
  6. md5_string_unicode
  7. md5_file
  8. sha1_string_utf8
  9. sha1_string_unicode
  10. sha1_file

NOTE : 编码不是加密！一个base64编码呈现肉眼无法读取该文件，并代表解码的用户将需要做出努力，但它并不安全在黑客的情况下。建议您这些混用函数，用自己的加密（加密形式有很多脚本是在互联网上提供）。
