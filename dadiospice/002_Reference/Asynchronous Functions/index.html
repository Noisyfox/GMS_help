Asynchronous Functions 异步函数

GMS有一些函数允许异步通信

在处理异步函数(指从现在开始作为异步函数)在GMS或许你想知道单词“异步”含义!嗯,实际的字典定义异步是:

"pertaining to a transmission technique that does not require a common clock between the communicating devices; timing signals are derived from special characters in the data stream itself."

在GMS环境下,一个异步函数,发送到一个web服务器,或者要求用户输入,甚至流数据,而让GMS继续运行而不阻塞。当收到信息回调一个特殊事件 触发异步,这反过来,允许任何实例与异步事件分配给它执行进一步的功能(虽然这不是必需的)。所以,基本上,这是一个沟通方式与一些外部资源(服务器、用 户或设备)没有游戏或项目阻塞等待一个回复。

为什么这个很重要?嗯,因为它允许GMS继续运作而发送或接收数据,这意味着你可以做很多事而不需要在游戏循环中 -

  • 不用玩家等待载入就可以从您的游戏设备中载入流数据
  • 当收到正确的连接信息才执行事件
  • 通过web服务器通信和交互数据

这一切使一切函数在一个更加流畅和不唐突的方式下。应该指出的是,异步事件将触发所有实例,有他们,所以你可以使用一个异步函数在一个实例中,并有异步事件处理响应另外一个,甚至其他的程序。你可以去这里了解更多信息的异步事件的工作- Advanced Use : Asynchronous Events.

  2. http_post_string

Cross Domain Issues 跨域问题

请注意,上面的http功能可能无法按照预期由于跨域安全问题。这意味着请求到服务器或尝试加载资源从跨领域都被封锁了,可能会出现返回空白结果或 404错误。的一个方法你可以绕过这个是有一些服务器端PHP允许某些域名来访问你的服务器(这也是保护你的资源和块服务器,不包括在PHP允许列表)。

下面是一个示例的代码可以使用:

$http_origin = $_SERVER['HTTP_ORIGIN'];
if ($http_origin == "http://127.0.0.1:51268")
   {
   header('Access-Control-Allow-Origin: *');
   }


Dialogues 对话

以前版本的GameMaker有弹出对话,中断了运行而等待响应的用户。然而大多数现代设备不像这样,将运行停止作为一个错误并关闭应用。为了防止 这种GMS这些异步函数等同于以前的版本那些功能,但不阻止游戏运行在任何时间,这意味着当称为游戏正常运行时的背景用户响应显示对话。一旦用户回应,一 个对话框Dialogs Asynchronous Event将被称为异步事件,这样您就可以处理结果。

下面是一个列表的所有可用的异步对话函数：

  1. get_login_async
  2. show_message_async
  3. show_question_async
  4. get_string_async
  5. get_integer_async
