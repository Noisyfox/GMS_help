In App Purchases 应用程序内购买

本节讨论在应用程序内购买你的游戏。

在App中购买是一个现代的重要组成部分，跨平台的游戏，允许你的游戏是免费的“免费增值”的内容，但某些内容是只能通过从游戏本身的购买。例如，您的益智游戏是免费的，但额外的关卡要花费用户0.99美元。这个应用程序模型是很常见的，可以设置工作的最低金额在 GameMaker:Studio中，本节的手册中有详细的所有可用函数，使您能够做到这一点，以及一些建议您的配置自己的服务器和如何跟踪和处理不同的平台上进行采购。

NOTE: 此功能仅限于专业版 Professional Version of GameMaker:Studio，只有那些版本包含iOS，Android或Windows 8的目标模块。

入门

在GameMaker:Studio 中设置您的游戏在应用程序内购买相对来说还是简单的，但它仍然需要一定量的知识的目标平台的搭建以及周密的计划，然后再开始创建你的游戏，如果您是刚刚开始使用GameMaker:Studio，或者您不确定怎样可以在 Android, iOS or Windows 8 应用商店设置，那么我们不建议您使用该功能。

在做其他事情之前，你必须确保全局游戏设定中的在应用程序内购买是开启的，在您使用这些功能之前。这节还允许您链接到自己的配置服务器控制可以购买的其他东西（更多信息请查看  Global Game Settings - In App Purchases Tab)）。

为了帮助那些你以前从未使用过任何类型如IAP服务，并解释IAP不同的方式，可以在不同的设备上进行测试，请参阅以下页：

  1. IAP 示例
  2. IAP 测试

NOTE : GameMaker:Studio 当前每个标题允许购买的应用不超过16个!

WARNING : 将无法在进行购买时使用的GameMaker IDE.调试标题。相反，你将需要创建最终的可执行文件，以确保标题能正确地沟通，与苹果商店，谷歌玩家或微软应用服务器。

GameMaker:Studio 函数

下面列出的函数都集成在GameMaker:Studio的 iOS和Android的目标设备（使得能从您的游戏购买额外的积分、道具、关卡或者其他），这些有效函数是：

  1. iap_activate
  2. iap_status
  3. iap_store_status
  4. iap_acquire
  5. iap_consume
  6. iap_event_queue
  7. iap_is_downloaded
  8. iap_is_purchased
  9. iap_product_status
  10. iap_product_details
  11. iap_product_files
  12. iap_files_purchased
  13. iap_restore_all

对于那些为应用程序购买，你拥有的其中存储着供下载的包服务器，你可以找到通过GameMaker:Studio如何将其设置为工作基本信息：

  1. IAP Server Setup 服务器安装
