IAP Example

This section gives a simple example of how to setup IAP in your games.

For the purpose of this example, we will use the Android target module as that is the simplest to set up and use, but the general outline given here is more or less the same for iOS (Windows 8 is more complicated. Please see the Windows 8 section of the main In App Purchases page). Note that this section deals with client side purchases only... for server side information please refer to the page on IAP Server Setup.

Setting Up The Store

Since testing your app in real-time is essential for IAP to work correctly, it is recommended that you upload a version of your game to the Google Store as soon as possible for testing. You can even make a specific "test app" as long as you make sure you use the same android package URL as you intend your "real" game to have (ie: com.MacSweeney Games.CatchTheHaggis), or the iaps won't be found by your real app later. The full testing process would go something like this:

  1. Go to the Global Game Settings - Android Tab and take note of your app id (ie: com.MacSweeney Games.CatchTheHaggis) and have a look at your release version. Note that it automatically increments each time you compile an .apk, so you should be aware that this app id must be the same as (or greater than) the app id that you publish on google play.

  2. Now go to the Global Game Settings - In App Purchases Tab and enable in app purchases. You need to put your Android public key that you get when you register your developer account on google play in the space provided too.

  3. Upload the *.apk (with In App Purchasing enabled) to the store.

  4. Give to your IAP a unique iap id. This can be anything, but whatever you decide, try and keep it consistent throughout your apps, as it will make tracking purchases and adding them in later much simpler. For example, you can use a simple numeric code for each purchase: 1000, 1001, 1002, 1003 etc...

  5. Ensure that the billing information for your Google Checkout Account is filled in and has been tested and approved, otherwise you can't proceed to use In App Purchases.

It is worth remembering that you can't test IAP using the YoYo Runner. So you will have to create the final *.apk (using the Create Executable option from the IDE) or, if you wish to test on multiple devices, it is suggested that you use some third party solution (like DropBox) to compile your .apk file and open it on your test devices via wi-fi.

There are two basic types of in app purchase: Consumables, and Non-Consumables, with the fundamental difference between the two being that the first one can be bought multiple times, while the second can only be bought once.
