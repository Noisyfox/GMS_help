iap_activate

Enables support for making in-app purchases.

Syntax :

iap_activate(product_index);


ArgumentDescription
product_index Either a string (to query custom server) or a real number (of a ds_list index).


Returns : N/A


Description

This function enables support for making in-app purchases and prepares GameMaker:Studio by supplying the product server index for your game, or a ds_list that holds a series of ds_maps with the information on each available purchase. If you are using a proprietry product server, the product index should be as a string and use the reverse URL format that is common to device packages, eg: "com.MacSweeney Games.CatchTheClown", and this will be used to query your server for the set of available purchases for the game (for further information on setting up a proprietry server, please see the section Global Game Settings - In App Purchases Tab.

If you are not using your own server to store the available purchases for your game, then you will need to tell GameMaker:Studio all the available purchase options by setting them in individual ds_maps, which are then stored within a ds_list. The individual purchase maps that are to be stored in the ds_list should have the following format of key-value pairs:

  • "id" - The Apple Store/Google Play product id for the purchase, eg: "LevelPack".

  • "title" - The name of the purchase, eg: "Level Pack 1".

  • "description" - The description of the purchase, eg: "Level Pack 1 for Catch The Clown".

  • "price" - The price of the purchase, eg: "$1.00".

  • "content_url" - The URL from which the content will be downloaded from (this must be a ZIP file with a flat hierarchy ie. no sub-folders), eg: "ExtraLevels2.zip"

  • "local_path" - The local path within the working directory where the purchased files are stored (Do not use leading or trailing slashes), eg: "ExtraLevels/1".

  • "type" - Windows Phone Only! The type of purchase which should be either "Consumable" or "Durable". "Durable" is the default value (non-consumable).



NOTE : All the key/value pairs that comprise a purchase map are strings!


Example :

var purchaseList, purchase1;
purchaseList = ds_list_create();
purchase1 = ds_map_create();
ds_map_add(purchase1, "id", "LevelPack");
ds_map_add(purchase1, "title", "ExtraLevels1");
ds_map_add(purchase1, "description", "Level Pack 1 for Catch The Clown");
ds_map_add(purchase1, "price", "$1.00");
ds_map_add(purchase1, "content_url", "ExtraLevels2.zip");
ds_map_add(purchase1, "local_path", "ExtraLevels/1");
ds_list_add(purchaseList, purchase1);
iap_activate(purchaseList);
ds_map_destroy(purchase1);
ds_list_destroy(purchaseList);

The above code will create a ds_list and a ds_map, which is then populated by the information for making a purchase. this map is added to the list and then actiavted as an available purchase for the game. finally the ds_list and ds_map are removed to prevent memory leaks as they are no longer needed.
