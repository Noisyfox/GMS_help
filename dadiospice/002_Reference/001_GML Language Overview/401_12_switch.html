The "Switch" Statement

这个章节描述了如何使用 "switch" 

在许多情况下,您想让您的实例完成一个行动取决于一个特定的值。您可以使用一系列连续的 "if" 语句但当可能的选择得到以上两个或三个通常使用的 "switch" 语句更容易。switch语句具有以下形式:

switch (<expression>)
{
case <expression1>: <statement1>; ... ; break;
case <expression2>: <statement2>; ... ; break;
...
default : <statement>;
}


这种工作方式如下:

  • 第一个表达式被执行。
  • 接下来是不同的表达式在每个case语句后。
  • 一直执行从第一个case语句后正确的值,直到遇到一个break语句。
  • 如果没有case语句,执行之后继续默认语句(它没有默认声明,在这种情况下不会采取任何行动)。

请注意,多个case语句可以放在同样的声明。同时,中断不是必需的,如果没有break语句执行，代码将继续下一个case语句。这意味着您可以创建一个“switch”的等级顺序的不同部分的代码运行取决于输入值。这是一个典型的“switch”例子:

{
switch (keyboard_key)
   {
   case vk_left:
   case ord("A"):
      x -= 4;
      break;
   case vk_right:
   case ord("D"):
      x += 4;
      break;
   case vk_up:
   case ord("W"):
      y -= 4;
      break;
   case vk_down:
   case ord("S"):
      y += 4;
      break;
   }
}


上述代码使用了“switch”,以检查键盘事件,然后比较。如果它符合其中任何固定值然后对应的代码被执行。注意如何在代码中我们使用的方式, “switch”可以查看多个case，如果没有中断则继续检查各种key，也能获得相同的结果。其中的一个方法就是你可以允许多个配置运作在你的游戏 中。
