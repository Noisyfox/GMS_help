GML Overview 概述

GameMaker:Studio 有自己的编程语言称为 GameMaker Language (缩写为 GML).

GameMaker:Studio 的编程语言, GML, 比使用拖放的动作给与您更多灵活性和可操作性，您使用这种语言设计游戏时请注意他们的不同：

  • Scripts. 脚本是使用GML编写的简短代码，您可以通过自定义函数来控制和使用。

  • Events. 您可以使用代码插入到对象事件中来控制这些对象的实例的行动和交互。您甚至可以拖放动作模块来完成。

  • Room Creation Code. 每个房间都有一个独特的“创造事件”。您可以按照需要增加代码，它将在房间每次进入时运行。

  • Instance Creation Code 和您在对象中插入事件一样，您能够增加代码到单独实例中并放入到房间中。 这些代码将在对象事件创建前或者后（默认）运行（这能够在全局游戏设定中设置）并根据实例不同而不同。因此您可以有多个同个对象的不同实例，做出属于自己特殊的设计。

在本章节中我们将描述GML函数的基本结构和怎样使用，请注意开始之前您应该知道一些 GameMaker:Studio 的常识，首先您的全部资源（精灵，对象，声音等...）必须有命名，命名头为字母，命名只能由字母、数字和 '_' 来组成（例如"spr_Fish"是可以的，而 "spr Fish" 或 "6fish" 则错误），另外不能使用GML的函数名作为资源命名。另外请确认全部的资源都有不同的命名（在Scripts 部分中有IDE能够帮助您检查 ）， 您也可以使用 self, other, global, 或 all 这些特殊命名在语言中。
