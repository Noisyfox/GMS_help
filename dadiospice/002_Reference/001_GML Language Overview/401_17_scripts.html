Scripts 脚本

脚本是GML的重要部分，本章节讲解了如何使用。

当你创建一个脚本,您想要访问的参数传递给它,要么当使用脚本动作,或者当调用脚本作为一个函数从一个程序(或者从另一个,甚至相同的脚本)。这些 参数都存储在变量argument0,argument1,……,argument15。所以最多有16个参数。(请注意,从一个动作上调用脚本,只有前 5个参数可以指定)。您还可以使用参数[0…15)来引用传递给脚本的参数是一个数组。

脚本还可以返回一个值,这样他们可以用在表达式。为达此目的使用return语句:

return <expression>


应该指出的是,脚本的执行结束返回语句,这意味着任何代码之前返回将不会运行。这里是一个简短的示例脚本称为“scr_sqr”的平方计算传递给它的任何值,它包括错误捕捉以防它传递的参数不是一个实数:

{
if !is_real(argument0)
   {
   return 0;
   }
else
   {
   return (argument0 * argument0);
   }
}


在一段代码调用一个脚本,采取与调用函数时同样的方式,即写脚本名称与参数值在括号中。所以,上面的脚本将调用方法如下:

if keyboard_check_pressed(vk_enter)
   {
   val = scr_sqr(amount);
   }

它有时是必需要让你知道有多少参数,使得脚本正常调用它的代码,所以GMS还提供了如下只读变量:

    argument_count



更多脚本信息请见 Advanced Use - Scripts.
