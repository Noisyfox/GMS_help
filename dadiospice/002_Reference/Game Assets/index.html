Game Assets 游戏资产

本章节包括所有相关游戏资源的函数。

在 GameMaker:Studio中，您可以定义不同类型的游戏资产（或资源），像精灵，声音，字体和背景各资源类型都有它 自己可供选择的代码，用于修改它们，使用它们，以及使它们在你的游戏中移动。甚至允许你在运行时加载和保存这些资源到你的游戏，但警告您（改变资源很容易 导致严重的错误，在您的游戏里，因此，改变资源时遵循以下规则是一个好主意：

  • 不要移除（删除）在当前的房间正在使用的资源，或正在被任何一个房间直接引用。

  • 如果你已经改变了你的游戏插入的资源（例如，你已经取代了精灵与加载的图像），当您重新启动游戏使用的 game_restart() 函数，改变资产不会恢复到原来的形状。在一般情况下，当您操作插入资源时你不能再使用动作或函数重新启动游戏。

  • 从一个文件中改变资源是一个相对缓慢的过程，所以尽量不要在游戏中期使用，而是将其保存的时刻，放在在任何的暂停不会中断游戏的时候。

  • 在 游戏中创建资源（尤其是精灵和背景），很容易使用大量的内存，所以要小心这一点。例如，如果你有一个32帧的128?28动画精灵，你决定创建36个旋 转的副本，它会使用36x32x128x128x4 = 72 MB的内存！而且，这些资源不包括任何预先制作的纹理页面及原有的资产，它们包括一个额外的页面用于交换，这意味着有许多这类资产在下下来时很慢。

  • 请确保您删除不再需要的资源，否则，您就导致内存泄漏，这会减慢，最终使得您的游戏崩溃

通常。您不应该尝试在游戏运行过程中改变资源，最好是在游戏开始时或者房间开始前创建或改变资源。

在如下页面可以找到关于资源的信息：

  1. Sprites 精灵
  2. Backgrounds 背景
  3. Sounds 声音
  4. Fonts 字体
  5. Paths 路径
  6. Timelines 时间线
  7. Rooms 房间

有两个额外的关于处理游戏资源的函数。这些函数能被用于找到唯一索引，对于任何给定的资产价值和资产类型，它为从字符串动态创建游戏资源打开大门（例如）：

  1. asset_get_index
  2. asset_get_type
