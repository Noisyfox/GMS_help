Surfaces 表面

本章节讨论关于创建和操作表面的函数。

在某些情况下，您可能不希望直接绘制到屏幕上，而是要“画布”，再后来被操纵和绘制到屏幕上，在需要的时候。这样的画布被称为一个表面，表面可以是一个非常强大的图形工具在制作GameMaker:Studio游 戏时，。例如，您可以使用表面“捉”实例，它可以被摧毁，并以这种方式，你可以创建一个贴花效果显示实例的精灵在表面上，如果它仍然存在，允许绘制效果， 如碎片，血液等.. 没有任何实际的处理开销。另一件事你可以做表面纹理进行操作，或创建精灵“在飞”，或创建复杂的叠加。事实上，表面用途上是无穷的！

表面是很容易使用，但也有几个应遵循的基本规则，当您使用它们：

  • 首先，你应该明白，表面上是“不稳定”。这意味着，如果设备或窗口失去焦点或最小化（很好的例子是，当屏幕保护程序出现在Windows或Android 设备上时，应用程序失去集中呼叫）的表面可能会被破坏。这是因为它是存储在vRAM中，当目标平台需要其他的东西写入内存时，这意味着，你应该始终有某种 类型的故障安全的代码在恰当的位置，通常是使用 surface_exists 。

  •  其次，你应该注意的是表面需要使用大量的vRAM，所以你应该尝试让他们尽可能的小。通常情况下，你将尽量保持他们不超过视角或者显示窗口大小。

  • 第三，你应该尝试绘制表面在绘制事件中。这并不总是可能的，但由于GameMaker:Studio 绘制到屏幕上的优化的方式，它是建议你把所有的绘制放在绘图事件中。

  • 第 四，当手动绘制表面，表面总是在房间内的（0,0）的位置处。这意味着你可能需要绝对坐标转换到局部坐标系的表面。例如，如果你有一个视角尺寸的表面，想 画的东西在目前的该表面上，你应该从实际的x和y坐标减去视角的X和Y坐标，获得表面（ 0,0）的位置的相对位置。因此，代码看起来像这样：

    if view_current = 0
       {
       surface_set_target(surf);
       with (obj_Effect)
          {
          draw_sprite(sprite_index, image_index, x - view_xview[1], y - view_yview[1]);
          }
       surface_reset_target();
       }
    else
       {
       draw_surface(surf, 0, 0);
       }

表面的基本用法如下-首先创建一个表面，把它的索引分配给一个变量。然后，您可以设置绘图目标到表面而不是显示器，可以绘制您希望绘制的以及进行任何其他 操作。一旦你重新设定绘图目标，就使所有进一步再次绘制在屏幕上。有一点要注意的是，如果您需要整个显示表面（包括瓷砖，背景等），你应该分配到一个视角 使用的变量 view_surface_id[0..7] 以这种方式，在该视图中可见的将被绘制到相应的表面。

存在以下函数处理表面：

  1. surface_exists
  2. surface_create
  3. surface_create_ext
  4. surface_set_target
  5. surface_reset_target
  6. surface_copy
  7. surface_copy_part
  8. surface_get_height
  9. surface_get_width
  10. surface_getpixel
  11. surface_free
  12. surface_save
  13. surface_save_part

这些函数是专门用于创建和操纵表面，但要真正把它们画到屏幕上，你应该使用特定的绘制函数，可以在如下章节找到相关内容 Drawing Surfaces 。
