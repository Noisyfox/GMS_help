Shaders 着色器

一个着色器是两个一组的小脚本，它可以让你直接操作图形流水线并且改变图像绘制的方式。

A shader is a set of two short scripts that permit you to tap directly into the graphics pipeline and change how things are drawn.

着色器是非常强大的工具。你可以用它控制你游戏绘制图像的方式，并且成效很快，比如你可以调整精灵的色调或者产生一个全屏的扭曲特效。

Shaders are a very powerful tool that can be used to manipulate the graphics that your game renders to the screen, permitting incredibly fast effects that can range from, for example, adding a subtle colour hue to a sprite, right up to full screen distortion effects.


What Is A Shader? 什么是着色器？

着色器大提上来说是两个一组直接在显卡上运行的代码片段，由于完全靠GPU来执行，因此代码执行的非常快并且不会消耗任何CPU处理周期。

一个完整的着色器由两部分组成：一个顶点着色器和一个帧着色器（又被称作像素着色器）。这两段程序协同工作来控制显卡的渲染过程。这样你就可以实时的控制渲染时每一个像素缓冲的位置、颜色和透明度。

关于着色器的工作方式以及在渲染管线中所在的位置的完整概述已经超出了本手册的范围，但是 YoYo Games 推出了一系列 技术博客 供你深入了解着色器：

A shader is basically a two-part program that runs directly on the graphics card itself, making it very fast since the GPU is doing all the work and freeing up CPU cycles for your game code.

The full shader is comprised of a vertex shader program, and a fragment shader program (also known as a pixel shader). Both of these tiny programs work together in order to manipulate what the graphics card renders to the screen. This then permits you to manipulate in real time, the position, colour and alpha values that are actually rendered into the display buffer.

The Shader Editor 着色器编辑器

着色器编辑器几乎和脚本编辑器功能一模一样，也有自动完成、查找和替换、打印等，连快捷键都是一模一样的。但是请注意它没有提供语法检查，因为着色器使用的是特殊的着色器语言，而不是GML。

The shader editor has the exact same functionality as the Script Editor, with auto-complete, search, print, etc... as well as all the same keyboard shortcuts. However, please note that there is no syntax checking, due to the code being written using the specific shader language and not GML.

由于着色器是由两部分组成的，因此代码编辑器本身被分割成了两个部分，可以用上方的标签来切换。 即使你只想使用像素着色器，你也必须创建一个“保持原样”的顶点着色器（当你新建一个着色器的时候会自动创建好一个“保持原样”的像素着色器和顶点着色器）。

着色器程序本身在默认情况下是使用一种跨平台的着色器语言 GLSL ES 编写的。使用这种语言可以保证你的着色器在大部分的平台上都能够正确运行。但是如果你只想在某种特殊平台运行，你也可以选择另外三种不同的语言。下表列出了所有支持的着色器语言以及对应的目标平台：

The code editor itself is split into two sections, with each one being available from tabs at the top. This is because you cannot create a shader without both parts. Even if you wish to only use the fragment shader you will have to have created a "pass through" vertex shader (by default any new shader being created will have a vertex and fragment pass through shader already coded for you).

The shader program itself is written in a shader language with the default, cross platform choice being GLSL ES. Using this language guarantees that your game will work correctly on all platforms that shaders are available on, but you can also choose to use one of three other languages if you are only targeting a specific platform. The available languages and their respective targets are shown in the table below:

Shader Language 着色器语言Target Platform 目标平台
GLSL ESAll target platforms except Windows 8 (JS)
HLSL9Windows 32 only
HLSL11Windows 8 (Native only) and Windows Phone
GLSLMac and Ubuntu (Linux)

值得注意的是在使用 HTML5 平台的时候你可以使用 GLSL ES 着色器语言，但是你必须在 HTML5 全局游戏设定 中启用 WebGL。

关于着色器函数以及如何在 GameMaker:Studio 中使用着色器的更多细节请参见 参考: 着色器 章节。

注意: 这份手册不会包括任何有关 Open GL 着色器函数和变量的内容，除非是只有 GameMaker:Studio 才能支持的特殊功能（你可以在 这里 找到详细的说明）。关于 GLSL ES 函数和变量的内容说明请参见 OpenGL ES Shading Language (GLSL ES) Reference Pages.

It is worth noting that you can use GLSL ES shaders on the HTML5 target platform but you must have enabled WebGL in the HTML5 Global Game Settings otherwise it will not work.

For further details on the shader functions and how they can be used in GameMaker:Studio please see the section Reference: Shaders.

NOTE: This manual will not cover any of the Open GL shader functions and variables, unless they are special ones that exist ONLY in GameMaker:Studio (which can be found here). For a complete overview of the available GLSL ES functions and variables please see the OpenGL ES Shading Language (GLSL ES) Reference Pages.
