Constants 常量

这一节讲述了如何定义常量使你能够在动作、脚本和代码中使用。

This section outlines how to define constants that you can then use in actions, scripts and code.

常量，正如字面意思所示，是用来表示一个不可变的值（实数、布尔型或字符串）所用的量。它和变量最大的区别就是常量一旦被定义，其值就不可以再被改变了，同时常量是一种全局的量，在所有的房间、对象和脚本中都可以调用，并且无论在哪里调用，都会是同样的值。GameMaker:Studio 中提供了许多内置常量，比如在按键事件里使用的 vk_ 系列或者是用来表示颜色的 c_ 系列。除此之外你还可以定义自己的常量。

要定义常量，点击主窗口顶部 Resources 下拉菜单中的 Define Constants（或者点击资源树图标），就会出现以下对话框:

A constant is, as the name implies, something that can hold a constant value (real, or boolean) or a string. This is different to a variable in that it can't be changed once defined, although it is global in scope as all instances in all rooms can use it, and it will always be the same value. There are a number of built-in constants within GameMaker:Studio, like the vk_ ones for the keyboard or the c_ ones that hold different colors, but you can also define your own constants to be used in your games too.

To define constants you chose Define Constants from the drop down Resources menu at the top of the main window (or click on the resource tree icons, as explained below) and the following window will open :

对话框的主体是一个常量列表，在一开始的时候这个列表是空白的，你可以在这里添加自定义的常量。只需要点击下方标记有 Add 的按钮就会创建一个新的、空白的常量，常量的 Name 项会被选中。你必须给你的常量起名字，并且和 GameMaker:Studio 里其它各种资源名称一样，常量的名称也不能有重复，同时也建议你使用特殊的命名方式让你能够与变量名相区别，比如说用纯大写字母命名，或者使用前缀或后缀。请注意常量名称中不能出现任何下划线以外的符号，以及不能以数字开头。

一旦你给一个常量起好了名字，那么你就要给它设置一个值。常量的值可以被设置成数字、函数调用或者是字符串。字符串必须用双引号括起来。如果你定义了一个函数调用，请注意它不能使用任何游戏中存在的东西的引用（比如当前房间、实例或是脚本等），因为这里的函数是在整个游戏最开始进行初始化的，在那个时候那些游戏中的东西还不存在呢。不过在这里你可以访问一些资源，比如说精灵或者是对象。

The main part of this window is take up with the list of constants, and will start off empty, ready for you to create your own. To do this you would click on the button marked Add at the bottom left of the window, which will then create a new, empty, constant with the Name section highlighted. You must give all your constants a name, and as with all things in GameMaker:Studio this name must be unique and it is recommended that you distinguish it in some way from other variables, for example, by using all capitals, or by using a suffix or prefix. Note, that you cannot use any symbols except the under-bar (_) and the constant cannot start with a number.

Once you have named your new constant you must give it a value. This value can be numeric, a function or a string. For a string it must be wrapped in quotes ("") and if it is a function, it cannot make reference to anything in the game (the current room, instances, or scripts etc...) as the constants are initialised before everything else, but it can however use resources like sprites and objects.

当你定义常量的时候，你可以用下方的按钮来修改、新增、插入或删除他们。Add 按钮会在列表的最后新增一个常量，Insert 按钮可以在当前选择的常量上方插入一条新的常量，delete 按钮顾名思义是用来删除选择的常量的。当然你也可以点击 Clear 来清除整个常量列表，在你点击这个按钮的时候会弹出一个确认对话框来确保你是真的想要这么做而不是手滑。除此之外还有 Up 和 Down 两个按钮让你上下移动当前常量，而点下 Sort 按钮，常量列表就会按照字母顺序进行排序。

最后，有两个标着 Load 和 Save 的按钮。通过这两个按钮，你可以载入你事先用别的编辑器定义好的或者其它游戏项目里的常量到你的游戏中。如果你在载入常量时列表中已经有其它常量存在，那么新载入的常量会被列在原来常量的后面。而导出的常量只是简单的储存在一个 *.txt 文本文件中，并且可以很简单的用任何可以编辑文本文件的软件（比如Notepad++）打开并修改。你甚至可以自己从头定义一个常量文件导入到你的游戏中。定义一个常量形式非常简单：常量名 = 值。

When you have defined a few constants you can use the buttons at the bottom of the window to change them around and add, insert or delete them. Add will add one at the end of the list, Insert will insert the next constant above the current one, and delete will delete the current one. You may also press Clear to clear the whole list of constants, which will open a prompt to ask if you are sure you really want to do that before deleting the whole list. Beside these button are two labelled Up and Down which simply move the constants up and down the list, and beside those is the Sort button which will put the constants into alphabetical order, with the ones starting with "A" at the top.

Finally, there are two buttons marked Load and Save. With these, you can load into your project constants that you have previously defined in an external editor, or that you have saved from a previous game. If you have already defined constants when you load some from a file, the new ones will be appended on at the end of the list. A saved constant list is stored in a simple *.txt file and can easily be edited and added to by any application that reads these files (Notepad++, for example). You can even pre-define your own text file with a set of constants to be loaded into GameMaker:Studio as the format is simply : NAME = VALUE.

The Resource Tree 资源树

常量也是IDE左侧资源树的一部分。当你新建一个游戏项目的时候，你会有两类常量- 全局配置 和 默认配置。在“全局设置”里面创建的常量会在你所有的游戏配置中生效（更多细节请参见 配置 章节），然后你可以为其中的某些配置添加特有的常量，每一个配置都拥有单独的常量列表。

Constants are included as part of the resource tree on the left of the GameMaker:Studio IDE. When you create a new game, you will have two sets of constants - All Configurations and Default. Constants created in "All Configurations" will be available to you no matter what configuration you have chosen (see the section on Configurations for more information), and you can then add configuration-specific ones to any other lists that you will have, with each new configuration creating a new constants list.
