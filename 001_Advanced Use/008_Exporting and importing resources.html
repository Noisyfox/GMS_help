Importing Resources 导入资源

这允许你从一个游戏中复制资源到另一个游戏中。

This allows you to copy resources from one game to another.

在前几个版本的 GameMaker 里从一个项目中导入和导出资源到另一个项目中是一件很复杂的事情，并且如果你一有不小心，那么等待你的可能就是一个损坏的项目或者是数据丢失。 好消息是，在 GameMaker:Studio 中一切都不再是问题了，新引入的 *.gmx 文件让你在项目间共享资源成了一件轻松的差事。

当你保存你的项目时， GameMaker:Studio 会在项目文件夹（这个文件夹可以很容易的在 Help Menu 下拉菜单中找到）中创建多个以资源类型和 *.gmx 为后缀的特殊文件（比如一个房间资源可能命名为 game_room.room.gmx）。这些 *.gmx 文件包含了 GameMaker:Studio 用来定义一个对象、声音或者任何其它类型资源所需的一切信息。你只需要在另一个项目的资源树上右击并选择 Add Existing Resource，然后在弹出的文件选择对话框中选择你想要导入的资源文件即可把这个资源添加到其它的项目中。

你可以一次性选择多个文件同时导入到 GameMaker:Studio 中，但是前提是这些资源必须拥有相同的类型后缀（举例来说你可以一次性导入10个 *.object.gmx 文件，但是不能将一个 *.sound.gmx 和一些 *.sprite.gmx 混在一起导入）。同时需要注意的问题是，如果你试图导入一个对象资源到你的游戏里，那么这个对象对应的精灵不会被自动导入进来。你必须要自己确保对象所使用的各种精灵都被添加进游戏里了。同时你在导入一个房间资源的时候也要注意不要忘记把房间里用到的对象也一起导入进来。因此通常比较好的做法是先导入精灵、声音、字体和其它一些资源，然后再导入对象资源，最后再导入房间资源。

In previous versions of GameMaker it was a complicated business to import and export files from one project to another, and if you weren't careful you could end up with a corrupt game or loss of data. Thankfully, that has all changed with GameMaker:Studio as the *.gmx format makes sharing resources between projects a simple and painless procedure.

When you save your project, GameMaker:Studio creates a number of special files in the game directory (this can be found easily through the drop down Help Menu) with the extension *.gmx and a sub-extension that indicates the resource type (eg: a room resource would be called game_room.room.gmx). These *.gmx files contain all the information that GameMaker:Studio needs to define the object, sound or whatever resource, and you can add these into a new project simply by right clicking on the resource tree and selecting Add Existing Resource, which will open a standard file selector where you can browse to the file you wish to import.

You can select multiple files in this way too and they will all be imported into GameMaker:Studio, but they must all have the same sub-extension (ie: you can load ten *.object.gmx at once, but not a *.sound.gmx and a couple of *.sprite.gmx). It should also be noted, that if you add an object resource into your game, if it has a sprite associated with it this sprite will not be imported. You have to make sure that any sprites your object needs have been imported from their appropriate *.sprite.gmx. The same goes for any rooms that you import, as they will be missing the objects that they contained if you have not imported them previously. So, it is always best to import sprites, sound, fonts and other resources before objects, and those you should import before rooms.
