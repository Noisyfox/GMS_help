IAP Event

This is the sub event that will be triggered by any IAP call-backs received.

This event can only be triggered by the In App Purchases functions and will return some very specific results depending on the target module that is used as well as the functions being called. The following circumstances are those that will trigger this event:

  1. The store state changes.

  2. A product changes state:
    • On confirmation from iOS store.
    • On purchase.
    • On completion of downloaded files.

Along with this event there is a special function that can be called to get a ds_queue of values so that you can check to see if any of the above criteria have been met and react accordingly. For more information on this function please see iap_event_queue.

NOTE : The ds_queue created by the above function is only valid in this event, as it is automatically deleted at the end of the event by GameMaker:Studio.
