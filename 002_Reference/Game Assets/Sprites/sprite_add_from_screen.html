sprite_add_from_screen

Adds an area of the screen as a sub-image to a sprite.

Syntax :

sprite_add_from_screen(index, x, y, w, h, removeback, smooth);


ArgumentDescription
index The index of the sprite to add the new image to.
x The x position to copy from.
y The y position to copy from.
w The width of the area to be copied (from the x position).
h The height of the area to be copied (from the y position).
removeback Indicates whether to make all pixels with the background color (left-bottom pixel) transparent.
smooth Indicates whether to smooth the edges.


Returns : N/A


Description

This function works in exactly the same way as sprite_create_from_screen only instead of creating a new sprite from the area of the game window that you select, it adds the defined area of the screen as a new sub-image to a previously created sprite.

NOTE : This functionality is limited to the Standard Version of GameMaker:Studio and above.


Example :

spr_custom = sprite_create_from_screen(0, 0, 32, 32, false, false, 16, 16);
var i;
for (i = 1; i < 8; i +=1)
   {
   sprite_add_from_screen(spr_Custom, i, 0, 32, 32, false, false, 16, 16);
   }

The above code creates a sprite from the screen, assigning it's index to the variable "spr_Custom", and then uses a for loop to move across the screen and capture various sections which are added into the sprite as sub-images.
