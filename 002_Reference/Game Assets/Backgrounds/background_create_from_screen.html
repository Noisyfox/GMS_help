background_create_from_screen

Creates a background by copying the given area from the screen.

Syntax :

background_create_from_screen(x, y, w, h, removeback, smooth);


ArgumentDescription
x The x position to copy from.
y The y position to copy from.
w The width of the area to be copied (from the x position).
h The height of the area to be copied (from the y position).
removeback Indicates whether to make all pixels with the selected color (left-bottom pixel) transparent.
smooth Indicates whether to smooth the edges if transparent.


Returns : Real


Description

With this function you can create a background from an image on the screen. The screen is defined as the area of the game window where everything is displayed, so the x and y coordinates that you must input are relative to the (0,0) position of the game window (the top left corner). The width and height arguments are in pixels and define the width and height of the part of the screen to use.

Setting the "removeback" argument to true will remove a color from the background, by checking the bottom left pixel of the selected area for the color there and then using that as the color to be removed.

If you choose the "removeback" option, you may also want GameMaker:Studio to smooth the edges of the background by setting the "smooth" argument to true. All this does is create a semi-transparent border around the edges of the background after it has had a color removed.

NOTE : When you create a background in GameMaker:Studio with this method you must remember to remove it again (with background_delete) when no longer needed, otherwise there is risk of a memory leak which will slow down and eventually crash your game.


Example :

if keyboard_check_pressed(vk_enter)
   {
   if !background_exists(bck_custom)
      {
      bck_custom = background_create_from_screen(0, 0, 32, 32, false, false);
      }
   }

The above code checks to see if a background has been assigned to the variable "bck_custom" when a key is pressed. If the check returns false, a background is created form the screen and it's index is stored.
