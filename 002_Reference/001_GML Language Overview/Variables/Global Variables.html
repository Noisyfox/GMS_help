Global Variables 全局变量

本节介绍如何使用全局变量。.

一个基本描述全局变量,一旦声明,不属于任何实例，然而特别的是所有代码都可以访问。像局部变量，全局变量必须声明,但不像局部变量、全局变量始终 在内存中,直到游戏结束。因此,您可以创建一个全局变量来跟踪(例如)的子弹数量玩家然后就更新这个变量在不同的点在游戏中。一个全局变量不属于任何特定 的实例,可以访问、更改和所有实例所使用在任何时间,但任何变化的变量是“global”,在通常情况下使用该变量的都将受到变更的影响。让我们看一个例 子:

globalvar food;
food = 5;


所以,我们已经创建了一个新的变量称为“food”,我们已经宣布它作为全局。现在,任何实例可以使用和改变此变量在任何方式和所有其他实例将“看见”这个。例如我们可以有一个食物对象,球员发生碰撞,在碰撞事件我们有:

food +=1;


我们也有另外一个对象绘制这个变量：

draw_text(32, 32, "food = " + string(food));


基于全局变量,我们可以改变值和看到这些变化反映在的所有实例对象,引用这个变量。局部变量必须小心不要命名与您的全局变量一样变量在任何实例 中,会导致你的问题,让漏洞进入你的游戏!实际上,我们可以调用全局变量以另一种方式来帮助解决这个问题,这是通过使用单词“global”和“.“在变 量之前。下面的例子说明了这种用法:

global.food = 5;


通过这种方法,我们现在将不得不使用“global”在每次我们希望使用变量的时候 :

global.food += 1;

draw_text(32, 32, "food = " + string(global.food));


GMS有一组“内置“全局变量了,所以你应该意识到你有可能命名到与你的一个实例变量相同的或自己希望的全局变量却具有相同的名字，想知道你为什么越来越 错误!他们很容易被发现,然而,因为它们是在一个不同的颜色显示在代码编辑器中,也出现在自动完成框底部。大多数建在全局变量是非常具体的,只会被用在罕 见的场合,但是有三个特别的这是很有用的,你可以找到更多关于他们从下面列出的页面:

  1. score 分数
  2. health 健康
  3. lives 生命
