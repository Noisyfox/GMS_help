Variables And Variable Scope 变量与变量域

这里你能学习到在GameMaker:Studio中关于不同类型的变量。

如任何编程语言GML使用变量为大多数编程操作的基本单元。变量是用来存储信息，供以后（或即时）在设备的存储器中使用，并给他们一个名称，以便您可以参 考它们的功能和程序。在GML的变量可以存储一个实数（如100，2.456575，-56等..）或一个字符串（例如“hello world！”）。

变量是什么，我们用它来存储在一个或多个操作或函数使用的值 。想到“PI”，例如... 它是一个真正的世界的变量保存值是3.14159265（等..）。为什么呢？嗯，说“PI”比“三点一四一5 9 2 6 5”容易！因此，命名的东西让生活更简单了很多，这也意味着，变量应该是不断变化的值，我们无须改变无处不在的变量名，它仍然是相同的。在GML中，一个 变量的名称必须以字母开头，只能包含字母，数字和下划线'_'，最大长度为64个符号。因此，有效的变量是如fish，foo_bar， num1，和无效的变量类似6fish, foo bar, 或 *num.。现在，在许多编程语言，你需要“声明”变量，然后才能使用它。这基本上意味着，你告诉计算机你想使用名字，因此，它在内存中指定一块地方存储任何你决定保存到该变量的值。随着GML，并不总是必须的，因为它依赖于该变量的范围。当你的程序是GameMaker:Studio的时有四个主要的变量，每一个有他自己的范围（这可以被视为其运算范围，或延伸）。这些变量和它们的范围概述如下：

  • 实例 : 这是最常见的变量，并且在一个实例中所定义。这些到该实例是唯一的，并可以在该实例的任何事件和功能中使用。

  • 局部 : 这些变量必须使用的“var”函数声明。局部变量是仅在创建它的事件或脚本资源中有效。所以GameMaker:Studio 将创建变量，在事件中持续使用它，然后再“忘记”它，这意味着，如果你尝试在它失效后使用它，你会得到一个“未知变量”的错误。

  • 全局 : 一个全局变量是属于游戏世界本身，而不是任何一个实例。它必须首先被声明为全局，但之后的任何实例可以查看或改变该变量的值，该值将总是反映了上一次的操作，不管什么实例或代码框中都能执行操作。

  • 内置变量:  这些是特殊的变量“内置”在游戏世界中的对象和房间，他们可以是实例唯一或全局范围内的（但绝不是局部的）。
