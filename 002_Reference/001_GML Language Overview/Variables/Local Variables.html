Local Variables 局部变量

局部变量是我们首先声明的，然后才能使用和丢弃的。

局部变量是一个我们仅为一个特定的事件创建的，当事件已经完成只有丢弃(唯一的例外是在脚本资源,一个变量声明为var,然后丢弃)。为什么我们需 要它们?嗯,变量在内存中占用的空间,也许我们只是去使用它们的一个操作或功能在这种情况下,我们只需要在短时间内记忆,它的使用。这使您的代码干净整洁 以及保持内存空间优化真的需要它。声明一个局部变量我们使用函数var如下:

var i, num, str;
i = 0;
num = 24.5;
str = "Hi there!";


上面创建的所有变量将被“遗忘”(即:从内存中除去)事件(或脚本)结束时。你必须小心,这个名字你给var声明的变量不能分配给另一个实例变量 在对象运行代码,并确保你没有打算使用该变量的值存储在外部的事件声明它。这些变量在程序中使用了很多,特别是在循环迭代计算,或当使用一个值在一个操作 几次,不会再次重复。这里有另一对的例子:

var i = 0;
repeat (10)
   {
   inventory[i] = 0;
   i+=1;
   }


上面的代码创建了一个局部变量称为“i”,并将其设置为0,所有在同一行上。注意,在早期版本的GameMaker你不得不声明你的本地变量,然 后将它们分配给第一个值,但是在这个版本你可以声明并将它们分配给一个值在同一时间。上面的代码然后使用这个变量来初始化一个数组。除此之外随着变量 “i”不会被用于任何进一步的函数在实例,它可以是本地的范围。这里是一个例子:

var xx,yy;
xx = x - 32 +irandom(64);
yy = y - 32 +irandom(64);
instance_create(xx, yy, obj_blood);


我们在这里使用了本地变量来存储一些随机坐标,然后,我们可以用它来创建一个实例。在这个例子中你可以看到,它并不是必需的,我们使用这些变量但为了我们做的有可读性和易用性。如果我们使用此代码将使得我们做的更清晰和明确:

instance_create(x - 32 + irandom(64), y - 32 + irandom(64), obj_guts);


另一件关于var声明的变量应该注意…因为他们独特的事件,运行它们,它们可以用在任何其他实例通过代码!这意味着我们可以使用这些变量设置和改变事物在其他情况下使用“with()”构造(有部分在概述部分的GML手册)。实际的代码本身看起来像这样:

var num = instance_number(obj_Enemy);
with (obj_Enemy)
   {
   if num>10 instance_destroy();
   }


上面的代码是有效的,因为var声明的变量是局部的事件(或脚本),它包含的,不是实例,也不是游戏世界,所以可以用在任何函数在任何物体,只要它是在相同的代码块。
