Instance Variables 实例变量

实例变量是一个的范围仅限于创建它的实例。

一个实例变量中创建一个对象的实例,被认为是独一无二的实例——即:许多实例相同的对象可以有相同的变量,但每个变量能拥有不同的值是独一无二的每个实例。但一个实例变量是如何创建的?嗯,你创建新的变量只需指定值,见这个小例子 ：

potions = 12;
life = 100;
name = "Jock McSweeney";
strength = 5.5;
armour = -2;


正如您可以看到的,你只需要给这个名字,然后一个值(或数字或字符串)来设置该变量和准备好的内部使用的对象的一个实例。这些变量可以使用和修改以多种方式从实例中,例如这个代码可以在碰撞事件和以数量改变变量“life”:

life -= 5 + armour;


如果“life”是100,它现在将有一个值为97(100 -(5 + 2)= 97)。现在,这是一个简单的例子,你可以取代“armour”值为2,但是如果你在多个地方使用该值,然后再决定去改变它吗?你必须通过你所有的代码和 改变2到新值是费时的,非常容易出错!但如果你使用一个变量,你所要做的就是重新分配这一个新值,代码将自动使用新值从那时起,使事情更灵活和更容易修复 应该有问题。还应该注意的是即使一个值不会改变更容易记住一个变量称为“life”的手段,而不是只看一个数值!

GMS有一组“内置“全局变量了,所以你应该意识到你有可能命名到与你的一个实例变量相同的或自己希望的全局变量却具有相同的名字，想知道你为什么越来越 错误!他们很容易被发现,然而,因为它们是在一个不同的颜色显示在代码编辑器中,也出现在自动完成框底部。
