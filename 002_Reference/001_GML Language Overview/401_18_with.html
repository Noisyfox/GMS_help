The "With" Construction

这个章节描述了如何使用 "with".

如章节 Addressing Variables in Other Instances, ,它可以读取和改变变量的值在其他实例。但在许多情况下,你想要做更多的不仅仅是改变单一变量与其他实例。例如,假设你想把所有的球在你的游戏对象8像素。你可能认为这是实现简单地通过以下代码:

obj_ball.y = obj_ball.y + 8;


但这是不正确的,因为右边得到的值的是第一个球和y坐标增加8到它。接下来这个新的值被设置为y坐标的球,那么结果就是,所有的球都得到同样的y坐标,即使你使用以下:

obj_ball.y += 8;


它将会有完全相同的效果,因为它只是一个第一个语句的缩写。那么,我们如何实现呢?为了这个目的在GML存在着语句。其global形式是:

with (<expression>) <statement>


<Expression>显示一个或多个实例,为此,你可以使用一个实例id,一个对象的名字(这表明这个对象的所有实例运行代码块)或一个特殊的关键字(all, self, other)。<Statement>现在执行的每个实例声明,如果该实例是当前(self)实例。所以,移动的所有实例对象8像素球下来,你可以输入:

with (obj_ball) y += 8;

如果你想要执行多个语句,把周围的括号起来,这样你就像执行任何其他程序一样。比如,将所有球到一个随机的位置,您可以使用:

with (obj_ball)
   {
   x = random(room_width);
   y = random(room_height);
   }


注意,在statement(s),指定的实例已经成为目标(self)实例,运行代码块,这意味着语句原始实例(包含”with“和代码块)已经成为其他实例。比如,改变所有球的位置,您可以输入:

with (obj_ball) { x = other.x; y = other.y; }


with的是一个极其强大的工具,在很多情况下是有用的，所以重要的是你充分理解如何使用它。有几个例子使用如下:

with (instance_create(x, y, obj_Ball))
   {
   speed = other.speed;
   direction = other.direction;
   }


上面的代码将创建一个obj_Ball实例，并分配了的速度和方向的运行代码。

with (instance_nearest(x, y, obj_Ball))
   {
   instance_destroy();
   }


上面的代码将破坏一个obj_Ball最近实例的运行代码。

var inst;
inst = none;
with (obj_ball)
   {
   if str > other.str inst = id;
   }
if inst != noone target = inst;


上面的代码是比之前的稍微复杂由于它使用本地变量。这个变量的局部脚本而不是实例,所以可以使用和访问的所有实例在代码块中。所以,在代码中我们需要设定特别关键字 noone 并且使用"with"在每个obj_Ball实例中检查它们 "str"变量针对代码块中的实例。如果变量的值较大,那么他们唯一的id存储在“inst”局部变量,这意味着最后的代码,只有实例值大于调用实例(或关键字noone i如果没有较大的)将存储在本地变量 "inst"。更多信息见部分局部变量 Variables And variable Scope.
