Addressing Variables in Other Instances 寻址变量在其他实例中

在这里你可以找到各种方法来解决变量在其他实例。。

本章基于 variables ，您可在那章中找到如何创造和使用变量在单独的实例中，或者使用全局变量，但是如果您想让一个实例中变量传到另外一个实例，就不同了？很多时候您会想这样 做，例如子弹碰撞，您会找出碰撞到哪种子弹的伤害，或者您想停止移动所有的球来思考，或者您想移动主角到一个特定位置，或者其他您游戏中的特定情况。庆幸 的是GML已经具备这种机制来实现这种情况：

Using a point "."

一种常见访问或改变的变量在另外一个实例中的用法是用对象名后加上 "." 。例子如下：

obj_ball.speed = 0;


如上代码您设定速度到全部"obj_ball"的实例，因此这种方法使你能改变特定对象的全部实例，但是您如果只想设定速度到一个单独的球，而不是所有球？这看起来会更有难度，但是规则还是一样，只是把对象名换成特定的实例名，我们将使用到 instance id 在GMS中处理单独实例。

每个实例在游戏中被创造时都会获得一个 instance id ，当您把实例放入房间在房间编辑器中，鼠标指到这个实例，实例的id会显示在窗口下方，但是如果您通过代码创造的实例也会有一个独一无二的id。这个数值总是大于或等于100000，像这样的数值可以使用在点符号的左边。 Note :这些数值后面跟点会被理解为小数点，要避免这种情况请使用括号！

例子如下:

(100012).speed = 0;


最后，您能使用特殊的关键值或者其他实例中它们自己的id。在GMS关键值是一种特殊类型的变量，识别和解释都有特别的方式。存在的关键值如下：

KeywordDescription
self 执行该代码的当前实例。
other 其他实例包括在碰撞事件 或通过 with 函数得到的其他实例。
all 房间中全部活动的实例。
noone 在全部中没有的实例（这感觉很诡异，但是你将在说明书其他部分看到它的用处）


关于使用变量，一个简答的情况是instance id 在执行代码前使用，例子如下...

使用关键值去设定全部实例:

if instance_position(mouse_x, mouse_y, all) = noone
   {
   all.speed = 0;
   }


使用变量设定单独实例 :

var nnn;
nnn = instance_create(100, 100, obj_ball);
nnn.speed = 8;
nnn.direction = random(360);


这些都是有效的读取，改变和设定其他实例中变量的方法，因为点仅是一个操作符。 它需要一个值作为左操作数和一个变量(地址)，右操作数同样，并返回地址的这个特定的变量指定的对象或实例。所有的对象名称,特殊对象表示以上仅仅代表 值,这些可以处理类似的任何其他值。对象名称,特殊的关键字,和实例id也可以用于许多其他功能,因为它们实际上在程序编写时是视为常量。

Using "other"

other 特殊的关键字有两个不同的方法,它可以用来引用一个特定的实例: 当使用 with 函数 (解释请点 here) 或者使用在 collision event, 本章节将解释.

一个碰撞事件只能发生在两个实例中。当然,你可以有多个实例之间的多个碰撞，但他们都是基于解决两个之间的基础上，实例的碰撞事件和“other”实例相关的。所以假设你有一个角色对象,敌人对象和子弹对象，敌人可以对你开火。你可以指定每个敌人一颗子弹，但当创建时不同的伤害随机分配，例子如下：

var nnn;
nnn = instance_create(x, y, obj_Bullet);
nnn.damage = 5 + irandom(5);
nnn.speed = 8;
nnn.direction = point_direction(x, y, obj_Player.x, obj_Player.y);


看到我们如何使用点符号设置它的变量方法了吗？但是角色对象?它将如何检测到伤害？是通过使用 other 在碰撞事件，这是当然!

hp -= other.damage;
if hp <= 0 instance_destroy();


上面的代码将会扣除存储的值，在其他实例“damage”变量传到玩家“hp”变量，然后检查“hp”是否低于或等于0。如果它是那么破坏角色对象。请注意 other 这种使用方式只能在碰撞事件和其他的实例变量必须能被检查，否则会抛出一个错误。但是你可以为变量赋值，甚至创造出新的想法,像使用 other 的碰撞事件这样：

other.mana += 10; //add ten to the other instance "mana" variable
other.hit = true; //set the other instance variable "hit" to true, ceating it if the variable doesn't already exist
