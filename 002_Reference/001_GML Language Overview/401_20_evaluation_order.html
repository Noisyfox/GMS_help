Evaluation Order 计算顺序

本节讨论的函数和表达式的计算顺序。

当你使用GML游戏编程，你应该知道该函数调用的参数不能保证计算顺序。这意味着，顺序放置在你的代码中的函数计算顺序将随平台而改变，所以你应该 以明确的方式编码。这是由于不同的目标平台之间优化的差异，例如在Windows目标函数进行计算，由右至左，但的HTML5目标上，它们可以计算从左至 右。因此，为了避免你的任何问题，最好不要调用一个函数的参数中调用多种函数，因为你计算的顺序很可能依赖于环境。

看下面这个例子是什么含义，考虑下面的代码调用几个函数，并利用他们作为一个脚本的参数：

buffer_seek(buff, buffer_seek_start, 0);
scr_buffer_get_info(buffer_read(buff, buffer_s8), buffer_read(buff, buffer_s16),buffer_read(buff, buffer_s16));


在，这里的问题是，在某些平台上，被称为的最后buffer_read将第一个指定，所以所有参数的脚本将是错误的，你会感觉到因为正在读取的数据从缓冲 区“反向”命令。这里有连锁效应的影响进一步值的buffer_read函数，因此，所有的参数被传递给该脚本将是错误的！

为了解决这个问题，你应该明确地调用顺序所需的函数和存储返回的变量中的值，就像这样：

var val[0] = buffer_read(buff, buffer_s8);
var val[1] = buffer_read(buff, buffer_s16);
var val[2] = buffer_read(buff, buffer_s16);
scr_buffer_get_info(val[0], val[1], val[2]);


尽管它可能看起来更详细的方法，它使一切都清楚了，为了避免任何计算顺序可能出现的问题的。
