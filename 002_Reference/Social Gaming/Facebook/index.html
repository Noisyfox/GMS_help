Facebook

本节解释了所有Facebook的API函数。

目前Facebook在在互联网上的社交媒体中非常重要，许多游戏玩家和开发人员有他们自己的网页游戏和媒体发布。 GameMaker:Studio有大量的内置功能，允许您连接到Facebook和用户信息以及发表的图像和文字在墙壁和一些其他的东西。建议您访问 Facebook开发者页面 Facebook Developers ，然后再开始使用这些功能，以便有个一个坚实的了解，如何在Facebook绘制工作，以及提供必要的呼叫和权限，和为您提供的这些函数。

NOTE : 如果您的测试设备安装了Facebook的应用程序，这些函数将只在创建一个可执行程序时工作，，如游戏测试（使用绿色或橙色的“播放”按钮），将无 法正常工作。基本上，测试运行器和Facebook应用程序彼此沟通，这样的沟通机制，必须正确地设置，你只能通过创建应用程序按钮才能正确设置。

Android 应用程序

你建立你的Facebook页面接受一个Android应用程序，有一些重要的事情是要注意的。第一件事是，GameMaker:Studio Android的Facebook的函数目前不支持Android SSO 也不支持 Android Native Deep Linking  ，所以请让他们禁用。这也意味着，Android Package Name和Android Class Name 是不相关的，因为他们只需要本地连接，这意味着他们也可以留空白区域。

另外一节，您需要进入Android的密钥存储的哈希 Hash。由于这可以是相当困难的，我们已经包括了*.bat文件，将让你在Facebook教程自带GameMaker:Studio安装文件。该文件中可以找到的AppData文件夹 %appdata%\GameMaker-Studio\tutorials\8 - Facebook的连接和被称为 facebookkeydebug.bat。运行该文件之前，您必须在文本编辑器打开它，并改变指定的部分，使其为您的系统指向正确的文件，但是一旦你这 样做，并运行该文件，您将看到您的密钥库文件的哈希键，然后可以添加到Facebook页面（如果您使用多个密钥库，为每一个在这里你可以添加一个哈 希）。

iOS 应用程序

iOS应用程序的Facebook的基本设置与Andr​​oid的函数是一样的，但有几个关键的事情需要实际的Facebook开发者页面上为你的应用程序正确设置：

  • Bundle ID: 必须与你的游戏捆绑ID（通常形式是 "com.{yourcompany}.{yourgame}"）。
  • iPhone App Store ID: 这是与你游戏相关的Apple ID就像为iphone游戏找到iTunes的连接一样。
  • iPad App Store ID: 同样一样，只是用于 iPad.
  • Facebook Login: GameMaker:Studio 支持单点登录 Single Sign-On 为 iOS 应用, 让您可以让游戏从iOS Facebook应用中启用。
  • Deep Linking: 这是不支持的，请保持禁用。
  • URL Scheme Suffix: 这是不支持，请留空白。
