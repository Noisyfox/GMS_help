Data structures 数据结构

本节讨论不同,专业函数,您可以使用它来存储不同类型的数据。

在游戏中你经常需要将信息以精确的和有序的方式存储。例如,您可能需要存储的人物的物品清单,或你可能想存储网格的地方仍然需要访问。现在,您可以 使用数组,但是如果你想做更复杂的操作,比如排序数据或寻找一个特定的项目,你需要写大块GML代码可以执行,缓慢调试和真正的痛苦写!

为了解决这个问题,GMS有很多内置的数据结构,可以通过专业功能。有六个不同类型的数据结构可用,每一个有它自己的好处取决于类型的信息,你打算存储和你如何想操纵它后:堆栈、队列、列表、地图、优先队列,和网格。

从本质上讲,所有的数据结构以同样的方式工作,你创建一个数据结构和存储在一个变量其索引值。然后使用这个指标参考数据结构在所有进一步的函数调 用,可以执行它的操作。最后,一旦你完成你破坏数据结构又把它从记忆存储器。您可以使用尽可能多的结构的同时,你需要,所有结构可以同时存储字符串和实际 价值。

NOTE : 如同所有的动态资源,数据结构占用内存空间,所以应该总是被摧毁时,不再需要防止记忆的泄漏将会慢下来,最终崩溃你的游戏。

在不同的数据结构中可以找到以下部分信息：

  1. Stacks 堆栈
  2. Queues 队列
  3. Lists 列表
  4. Maps 地图
  5. Priority Queues 优先队列
  6. Grids 网格

在开始只用使用数据结构在您的游戏前,你也应该意识到,有些时候,由于舍入错误,你可能会得到一个不是你期待的结果。这可能是解决通过改变DS精度使用以下功能:

  1. ds_set_precision
