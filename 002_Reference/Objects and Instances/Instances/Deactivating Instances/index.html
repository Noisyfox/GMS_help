Deactivating Instances 停止实例

本章节关于停止实例的相关处理。

GameMaker:Studio为您提供的“关闭”实例的可能，使他们不再以任何方式处理。在技术上，他们真的不存在了，除了作为一个指 针在失活过程本身，这意味着，停用的实例不能被操纵，或以任何方式改变，直到它再次被重新激活。因此，这些函数应该非常小心，因为不正确使用它们可能会导 致问题，尤其是固体对象，作为一个固体对象已经停用不会被移动到下一个房间， 除非它被重新激活（从游戏中有效删除）。

通常没有必要在你的游戏每步中停止实例，这实际上可以使你的游戏落后，运行速度慢。相反，它是建议你只运行这些函数在（例如）计时器的每几步，或者如果视角改变位置，尤其重要的是，你不能在绘制事件中使用这些函数，因为这可能会导致你的游戏严重的错误。

NOTE : 停用的情况下，有物理活动并没有停止他们的装置物理模拟互动。为您激活/关闭的情况下，为此你应该设置他们 phy_active 变量为“ 真“或“ 假“就像你激活和停止实例一样。

停用实例存在以下函数： 

  1. instance_activate_all
  2. instance_activate_object
  3. instance_activate_region
  4. instance_deactivate_all
  5. instance_deactivate_object
  6. instance_deactivate_region
