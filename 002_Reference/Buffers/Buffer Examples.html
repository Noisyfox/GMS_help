Buffer Examples 缓冲区示例

此页面有两个例子，使用缓冲区，以帮助了解如何使用它们。

使用缓冲区制作检查点

一个简单的例子是如何使用缓冲区用于 的任何平台，关于函数 game_save_buffer。此函数是将当前的游戏状态的“snapshot”，并把它保存到一个预先定义的缓冲区，这样就可以读取从在这一点上再次载入游戏。

NOTE: 此函数是非常有限的，它是专为初学者得到一个检查点系统的快速运行，但更先进的用户可能更喜欢自己的系统使用的 File functions，由于这样的事实，该游戏将不保存任何的动态资源，您可以创建在运行时，如数据结构，表面，添加背景和精灵等..

我们需要做的第一件事是创建一个新的对象来控制保存和加载的，所以你会做一个并给它一个创建事件。在这种情况下，你可以将下面的代码：

SaveBuffer = buffer_create(1024, buffer_grow, 1);
StateSaved = false;

第一行创建了一个grow缓冲（因为我们不知道所保存的数据）的最终大小为1024字节，1字节对齐。然后创建一个变量核对，如果游戏已经保存（这将是用于加载）。

下一步，我们将增加一个按键事件（例如），我们将保存当前的游戏状态，创建的缓冲区：

StateSaved = true;
buffer_seek(SaveBuffer, buffer_seek_start, 0);
game_save_buffer(SaveBuffer);

上面会先设定控制变量设置为true（这保存游戏，当我们保存到缓冲区中），然后寻求的缓冲区开始之前把它写的当前状态保存。为什么我们要用 buffer_seek？嗯，如 Using Buffers 页，你读取和写入到一个缓冲区，数据被添加到最后一个位置。这意味着，如果你不设置缓冲区的开始，然后回来告诉我，当你保存你将加入在当前缓冲区到缓冲区中的数据读/写位置，所以我们使用的函数buffer_seek移动告诉缓冲启动。

现在我们已经保存了当前的游戏状态的缓冲区。下一步将是代码如何加载，可在另一个按键事件：

if StateSaved
   {
   buffer_seek(SaveBuffer, buffer_seek_start, 0);
   game_load_buffer(SaveBuffer);
   }

本游戏将被加载，在你上面的代码的末尾事件中。

NOTE: 这是仅用于在同一个房间，而不是生成完整保存的游戏已经关闭或重新启动您的游戏后！


添加到控制器对象的最后一件事是一个“clean up”的代码。缓冲器存储在内存中，因此，如果你不清理，当你与他们完成，你可以得到内存泄漏，最终将导致你的游戏崩溃。所以，你可能会添加一个房间结束事件（从其他事件类别）：

buffer_delete(SaveBuffer);

现在，这个对象可以被放置到一个房间里，并且按键从缓冲区保存和加载房态。

联网缓冲区

当使用“GameMaker:Studio”的网络函数，你必须使用要创建的缓冲区数据“包”，也就是通过网络连接发送。这个例子显示如何做到这一点，但由于网络的可能性的范围，它是唯一的设计展示了如何使用缓冲区本身，而不是完整的网络系统（演示文件包含GameMaker:Studio显示这个）。

我们将展示的第一件事情是创造和使用的网络连接的客户端的缓冲区。此缓冲区将被用于创建小的数据包，然后将其发送到服务器，以便在创建事件的一个实例，我们将分配一个缓冲区类似这样的：

send_buff = buffer_create(256, buffer_grow, 1);

我们使缓冲区小（256字节），它并不适用于持有大量的数据，我们使它成为一个glow缓冲，以确保没有错误的，我们需要添加更多的数据可以在任何时间发送，对齐方式设置为了方便。

现在，让我们说，我们希望我们的客户将数据发送到服务器。对于我们需要创建一个缓冲区“包”，在这个例子中，我们要发送一个按键事件，喜欢当玩家按下左方向键移动周围的游戏。要做到这一点，我们编写必要的数据到缓冲区，然后把它关闭：

buffer_seek(buff, buffer_seek_start, 0);
buffer_write(buff, buffer_u8, 1);
buffer_write(buff, buffer_s16, vk_space);
buffer_write(buff, buffer_bool, true);
network_send_packet(client, buff, buffer_tell(buff));

记录到缓冲区之前，我们已经设置了“告诉”开始的缓冲网络始终需要将数据从一个缓冲区的起始。然后，我们写的检查值（这将在所使用的服务器，以确定要处理 的事件的类型），则该键被使用，则是该键的状态（在这种情况下，真正的按下）。然后此缓冲区由网络功能作为一个数据包发送。请注意，我们不发送整个缓冲 区！我们只发送数据写入，使用的 buffer_tell 函数返回当前的读/写缓冲区的位置（请记住，写入到缓冲区移动“告诉”到底已经写什么）。

接收服务器上的数据是什么？接收到的数据的数据包必须被写入到缓冲区中的服务器上，然后用于更新的游戏。为此，我们将使用 Networking Asynchronous Event 在网络控制器的服务器对象，这个简化的代码如下所示：

var buff = ds_map_find_value(async_load, "buffer");
if cmd == buffer_read(buff, buffer_u8);
   {
   key = buffer_read(buff, buffer_s16 );
   key_state = buffer_read(buff, buffer_bool);
   }

异步事件将包含一个特殊的的临时ds_map（它的事件结束时自动被删除），其中包含不同的信息，根据从网络输入的数据不同的。在这种情况下，我们 假设了地图进行了检查，发现是一个缓冲区中的数据从客户机发送的数据包。我们现在的第一块缓冲区中的数据，看看有什么样的事件已被送往检查 - 在这种情况下，值“1”是一个关键的事件，但是编码这些东西时，你应该定义常量持有这些值简化的东西 - 然后将其存储被按下的键，其状态（真=按下，假=释放）。然后，可以使用此信息来更新所有的客户端发送客户端播放器的新状态。

NOTE: 在网络异步事件的结束后从ds_map创建的缓冲区自动被删除，因此在这里不再需要使用 buffer_delete 。
