Steam API 应用程序接口

本节说明 Steam API相关的函数。

本节是为已被Steam API 允许你游戏发布到该平台的这些用户。为了能够使用这些函数，您必须已被steam许可，通过一个出版商或通过审批制度，并正确设置 Steam Preferences 和 Steam Global Game Settings 。

检查Steam客户端/服务器

下面的一组函数都检查某些方面的Steam客户端或服务器API的可用性，并取决于服务器的因素返回true或false，如占线或用户在登录等..这意味着，这些函数应该用于任何其它函数之前调用，以确保你的游戏客户端/服务器的设置是正确的：

  1. steam_initialised
  2. steam_stats_ready
  3. steam_is_overlay_enabled
  4. steam_get_persona_name
  5. steam_get_user_steam_id
  6. steam_is_user_logged_on

Steam 覆盖

在Steam叠加带来的显示功能，能显示在Steam接口提供给用户的视觉。这通常是由用户自己使用的组合键，但你也有可能做这件事从你的游戏内，这样可以映射一个按钮或事件显示叠加，使用以下功能：

  1. steam_activate_overlay
  2. steam_is_overlay_activated

排行榜

Steam API支持持久排行榜自动有序的条目。这些排行榜可以用来显示全局和朋友的排行榜在你的游戏你的游戏社区网页上。每场比赛可以有多达10,000排行榜， 每个排行榜可以被检索立即后一个球员的得分已经被插入到它，但注意，每个排行榜，玩家可以有只有一个入口，虽然数量没有限制，玩家每排行榜。

每个排行榜条目包含一个名称为排行榜，得分和排名，这数据将被替换为用户创建一个新的排行榜条目时，可以使用以下函数添加和检索这些数据形成排行榜您的游戏：

  1. steam_create_leaderboard
  2. steam_upload_score
  3. steam_download_scores
  4. steam_download_scores_around_user
  5. steam_download_friends_scores

成就与统计

Steam的统计数据和成就的API提供持久性，漫游的成就和统计跟踪为您的用户提供一个简单的方法，为您的游戏。用户的数据与自己的Steam帐号，每个用户所取得的成就和统计数据可以格式化并显示在他们的Steam社区简介。

除了提供高价值的奖励你的游戏的玩家，成就是非常有用的鼓励和奖励的团队精神和玩家互动，游戏目标提供额外的维度，用户花更多的时间在游戏中奖励，并因此它是建议你的游戏的有几个。他们很容易从Steam仪表板，但会要求你为他们创造特殊的图标。

统计跟踪细粒度条信息，如播放时间，数字电源ups使用，等你可以选择使用它们只是为了跟踪内部游戏数据 - 这样一来，例如，您可以授予的成就的基础上多在多台计算机上的用户会话的游戏收集的统计信息。或者，你可以跟踪有趣的游戏数据显示，在用户的Steam社 区页面，用户可以在自己的统计比较，对他们的朋友。

三种类型，整数，浮点数和平均统计， GameMaker:Studio允许您修改每一个从你的游戏，但他们应该已经被初始化了Steamworks你的游戏控制。

NOTE: 您必须等待，直到返回 steam_stats_ready 为真，然后再尝试读取或写入统计数据和成就。


下面几页中描述的函数，进一步的细节和说明，如何使用统计数据和成果，可以发现：

  1. steam_set_achievement
  2. steam_get_achievement
  3. steam_clear_achievement
  4. steam_set_stat_int
  5. steam_set_stat_float
  6. steam_set_stat_avg_rate
  7. steam_get_stat_int
  8. steam_get_stat_float
  9. steam_get_stat_avg_rate
  10. steam_reset_all_stats
  11. steam_reset_all_stats_achievements

如果用户在离线模式下，steam不断使API可以正常使用的统计和成绩数据的本地缓存。无法提交任何统计数据都保存为下一次用户在线。事件已经有 超过一台机器上的修改，steam会自动合并取得的成就和选择的一组统计数据，有更多的进展。因为蒸汽保持统计数据的本地缓存，它是没有必要的，对于游戏 也保留磁盘上的数据的本地缓存，尤其是当这种缓存往往在冲突中和当他们这样做，它看起来仿佛一个用户自己的进步有已经回归了，这是一个令人沮丧的经验。

云函数

云steam 你的游戏提供了一个简单和透明的远程文件存储系统。所有文件写入到磁盘使用GameMaker:Studio的功能将被复制到服务器，在比赛结束后退出steam。然后，如果用户改变了电脑，文件将被下载到新的电脑游戏推出之前，这意味着该游戏就可以访问文件，通过使用适当的函数读取他们。 Steam客户端做的工作，确保在所有计算机用户可能访问的文件保持同步。

NOTE: 默认情况下，云不能用于在Steamworks上的游戏，它必须是提前来至Steamworks游戏管理员的“云”页面 ，在页面上你应该设置字节和页面配额。下次你发布你的游戏到Steamworks的配置，准备使用云存储。


以下函数可用于从 GameMaker:Studio访问Steam Cloud from

  1. steam_is_cloud_enabled_for_app
  2. steam_is_cloud_enabled_for_account
  3. steam_get_quota_total
  4. steam_get_quota_free
  5. steam_file_exists
  6. steam_file_size
  7. steam_file_persisted
  8. steam_file_write
  9. steam_file_write_file
  10. steam_file_read
  11. steam_file_share
  12. steam_file_delete

用户内容

Steam API集成在GameMaker:Studio 允许某些用户生成内容的共享。您可以为你的游戏使用这些共享文件为到Steam Workshop，和屏幕截图到用户文件夹。

  1. steam_publish_workshop_file
  2. steam_is_screenshot_requested
  3. steam_send_screenshot

DLC

Steam 支持免费和付费下载内容（DLC），Steam客户端，游戏内容下载作为一个单一的应用程序在用户的游戏列表，通过游戏属性对话框查看下载内容出现。一旦 拥有，被视为本游戏的一个组成部分和Steam下载的内容会自动更新一个补丁和安装的内容，当用户安装的游戏的内容时。

由于这是所有处理由Steam服务器和任何DLC的配置是通过了Steamworks控制面板，所以只有一些必要的函数，在GameMaker:Studio检查这个额外的内容：

  1. steam_user_owns_dlc
  2. steam_user_installed_dlc
