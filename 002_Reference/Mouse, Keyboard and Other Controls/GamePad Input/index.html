GamePad Input 手柄输入

本章节是关于您游戏中的手柄输入。

GameMaker:Studio 有一些专用函数，可被用于侦测模拟或数控的多个连接的游戏手柄。这些函数类似 Device Inputs,，在这你可以检测多达四个不同的游戏手柄连接和处理输入的每一个使用相同的功能

当游戏手柄函数工作时，有一些内置的常量可以用于获取/定义按钮和棒轴等的.. :

ConstantDescription
gp_face1顶部按钮1 （对应Xbox 360控制器的“A”或PS3控制器的叉）
gp_face2顶部按钮2 （对应Xbox 360控制器的“B”或PS3控制器的圆圈）
gp_face3顶部按钮3 （对应Xbox 360控制器的“X”或PS3控制器的方形）
gp_face4顶部按钮4 （对应Xbox 360控制器的“Y”或PS3控制器的三角形）
gp_shoulderl左肩键（数字）
gp_shoulderr右肩键（数字）
gp_shoulderrb左肩键（模拟）
gp_shoulderrb右肩键（模拟）
gp_select选择键
gp_start开始键
gp_stickl左摇杆按下（作为按键）
gp_stickr右摇杆按下（作为按键）
gp_padu方向盘上键（数字）
gp_padd方向盘下键（数字）
gp_padl方向盘左键（数字）
gp_padr方向盘右键（数字）
gp_axislh水平摇杆左（模拟）
gp_axislv垂直摇杆左（模拟）
gp_axisrh水平摇杆右（模拟）
gp_axisrv垂直摇杆右（模拟）

兼容性

下面的列表显示了当前跨平台的兼容性（注意，这将改变未来的更新）：

  • 完全支持Windows的一次允许最多4个连接的设备。

  • MacOS的完全支持一次允许最多4个连接的设备。

  • HTML5游戏只支持谷歌Chrome（理论上应该支持任意数量的设备）。此刻没有其他的浏览器都支持这样的游戏手柄，但未来的更新应该将此功能扩展到完整的，跨浏览器支持。

  • GAMEPAD的支持也延伸到iOS与ICADE柜。左轴映射到杆形控制器（虽然输入是数字的，模拟），四个“脸”按钮映射到机柜前面的按钮，四个肩部按钮映射到那些在机柜后部的。

  • Android 目前只支持NYKO控制器，但只有当他们设置ICADE模式，这意味着你将不得不到打勾的ICADE的在Android全球游戏设置选项。他们需要 API12级，他们的工作充分，应当指出，游戏制作：Studio将注册为连接您的设备已配对的蓝牙设备，不论它的实际连接。因此减速它会显示作为连接0 “槽”时，在ICADE模式和“槽”1，否则（假设只存在与你的Android设备配对1 NYKO）和基本寄存器占用两个插槽，除非你有一个ICADE也配对与您Android系统。
