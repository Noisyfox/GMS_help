iap_product_details

Add product details to a ds_map.

Syntax :

iap_product_details(purchase_id, ds_map);


ArgumentDescription
purchase_id This can either be the Apple Store/Google Play id string, an URL id, or an index into the available purchases ds_list.
ds_map The ds_map that will store the product information.


Returns : N/A


Description

With this function you can generate a ds_map containing a number of key/value pairs that describe the purchase. The following data is returned within the map:

  • "Status" - The current status of the product, can be:
    1. "Purchased"
    2. "Available"
    3. "Unavailable"
  • "id" - The product id for the purchase.

  • "title" - The name of the purchase.

  • "description" - The description of the purchase.

  • "price" - The price of the purchase.

  • "content_url" - The URL from which the content was downloaded from.

  • "type" - Windows Phone Only! The type of purchase which can be either "Consumable" or "Durable". "Durable" is the default value (non-consumable).

NOTE: The ds_map used is not created by the function, meaning that you must create it previously and remove it again from memory when not in use using the appropriate function.


Example :

var p_map;
p_map = ds_map_create();
iap_product_details(p_id, p_map);
if ds_map_find_value(p_map, "status") == "Purchased"
   {
   draw_text(x, y, "Title: " + string(ds_map_find_value(p_map, "title")));
   draw_text(x, y + 32, "Description: " + string(ds_map_find_value(p_map, "description")));
   draw_text(x, y + 64, "Price: " + string(ds_map_find_value(p_map, "price")));
   draw_text(x, y + 96, "Product ID: " + string(ds_map_find_value(p_map, "id")));
   }
ds_map_destroy(p_map);

The above code will creates a ds_map and then stores the purchase details for a previously stored purchase id. If the "status" key hold the "Purchased" value, it will then draw the purchase details.
