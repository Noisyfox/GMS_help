Compile Errors 编译错误

本节列出了所有可能的资产编译器的错误。

创建你的游戏时，它很容易忘记来声明一个变量或错误类型的字符串作为一个真正的等..这些错误将被捕获作为GameMaker:Studio被编译，将打开一个窗口给出的错误，以及发现错误，在下面的图片所示： 您可以点击所显示的文本行，你会被直接抛出错误的对象，然后你就可以使用消息内包含的信息，以查明确切位置发生错误的对象中。然而，有时这些错误可能会显得有点神秘，所以下面你可以找到所有错误的完整列表和简要说明他们是什么意思。

  1. >unknown function or script

    您已经引用一个函数或脚本，它不存在。这可能发生，当你误键入一个函数或正在使用的对象从游戏中删除脚本。

  2. >Unable to find script

    您已经引用的脚本，是不再被找到的资源。

  3. >Execute script action is not bound to a script

    你已经使用了“执行脚本”动作脚本不再存在。

  4. >Failed to parse action_execute_script() - incorrect number of arguments received

    脚本执行脚本的行动已经给出了错误的参数数目

  5. >Trying to set a read only variable

    您试图分配一个值（字符串或实际）是一个变量，它标记为只读。例如：
    phy_com_x = 12;
    这个错误是 phy_com_x 为只读的。

  6. >Constant is invalid here

    您已经在你的代码放在一个常数，是不恰当的代码

  7. >Malformed repeat statement

    >Malformed if statement

    >Malformed while statement

    >Malformed do statement

    >Malformed for statement

    >Malformed with statement

    >Malformed switch statement

    >Malformed break statement

    >Malformed continue statement

    >Malformed exit statement

    >Malformed return statement

    你的说法是不正确的结构。具体细节请参见有关章节： GML Overview

  8. >Keyword until expected

    您已经成立了“do... until”循环，但未能提供“until”的一部分 GML Overview - Do

  9. >If requires a then statement

    然后省略了if语句的一部分。请参阅 GML Overview : If

  10. >Statement in a switch MUST appear after case or default

    您已经添加函数到switch以外的case或默认函数语句。例如：
    switch(id)
       {
       if x == 12 game_end(); break;
       }

  11. >Break used without context

    你已经添加了一个break到你的代码中，他所在位置不需要或者无意义。

  12. >Continue used without context

    你已经添加了一个continue到你的代码中，他所在位置不需要或者无意义。

  13. >Calling a function that needs an instance and no instance is available

    你应该使用instance_ *功能时存在任何实例在运行代码的时候，会显示此错误。

  14. >Calling a function that needs an \"other\" and no \"other\" is available

    你会得到这个错误，当使用其他引用的一个实例，该实例不再存在，例如，在发生碰撞时你已经破坏了其他实例。

  15. >Argument naming error, {n} arguments but no reference found to {a}

    您所提供的一系列的参数到一个脚本，其中一个或多个不代表任何价值。

  16. >Wrong type of arguments for *

    >Wrong type of arguments for /

    >Wrong type of arguments for div

    >Wrong type of arguments for Mod

    >Wrong type of arguments for +

    >Wrong type of arguments for -

    >Wrong type of arguments for &&

    >Wrong type of arguments for ||

    >Wrong type of arguments for &

    >Wrong type of arguments for |

    >Wrong type of arguments for ^

    >Wrong type of arguments for <<

    >Wrong type of arguments for >>

    您提供了错误类型的参数（实数或字符串）的表达式。例如，如果你有这样的事情：
    if "Hello" div 4
       {
       //do something;
       }
    你会得到这个错误，因为你不能使用div的字符串。具体细节请参见有关章节： GML Overview - Expressions

  17. >Division by 0

    除以0，所以你得到这个错误应该GameMaker:Studio代码，尽量做到这一点，这是不可能的

  18. >Modulo by 0

    你得到这个错误模表达式应该尝试使用0作为其操作符

  19. >Cannot compare arguments

    这个错误告诉你，一个或多个表达式尝试操作字符串和实数，这是不允许的。例如：
    if "Hello" == 74
       {
       //do something;
       }
    这会给错误的，因为你不能在表达比较两个值，因为它们是不同类型。

  20. >Symbol } expected

    >Symbol { expected

    >Symbol ) expected

    >Symbol ( expected

    >Symbol : expected

    >Symbol , or ) expected

    >Symbol , or ] expected

    >Symbol ] expected

    缺少你的代码是从它的语法上面的符号之一。这个错误通常捕获的代码编辑器，并强调为您。

  21. >Assignment operator expected

    这个错误将被发出，当你已经省略从表达式的操作符之一。例如：
    if var1 var2 == var3
       {
       //do something;
       }
    上面的代码缺少操作符之间的“VAR1”和“VAR2”，这样会给这个错误，虽然通常会被抓到之前被编译的代码编辑器。

  22. >Unexpected symbol in expression

    你有一个在错误的地方内表达的符号（如/，*，+，等..）。代码编辑器的语法检查，这通常会被代码编辑器警告。

  23. >Only 1 or 2 dimensional arrays are supported

    GameMaker:Studio 只允许1或2维数组，你应该尝试创建任何较大，那么你会得到这个错误。欲了解更多信息，请参阅：GML Overview - Arrays

  24. >Variable name expected

    你已经书写一个表达式，缺少变量在未有任何提供时使用。例如：
    if obj_Control. = "Hello"
       {
       //do something;
       }
    由于转移“.”后，没有任何变量，你会得到这个错误。
