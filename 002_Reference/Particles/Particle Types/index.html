Particle Types 粒子类型

本章节关于粒子系统中定义粒子的函数。

粒子是通过使用粒子系统在游戏中创建的图形效果。您使用这些函数定义粒子可用于在任何你创建系统中，并且通常的程序是在开始游戏前将定义好的粒子存 放到全局变量中。这些全局变量能够在爆发，发射或新建粒子在任何时间在整个游戏中是使用。您还可以创建，使用，然后摧破坏粒子的在需要时，如记忆体问题， 或如你想有很多不同的类型，但只在您的游戏的特定的点。无论您选择，粒子是一种用途很广的资源！

定义粒子

粒子需要创建索引存储在一个变量，以便进一步粒子程序可以正确地访问它们。以下函数用于创建，检查，清除和销毁的粒子类型：

  1. part_type_exists
  2. part_type_create
  3. part_type_destroy
  4. part_type_clear

一旦你已经创建了你的粒子类型，你应该给它一个形状。这可以是64x64像素的精灵与alpha混合的形状，建于14个内置形状，或者你可以使用自己的精灵，它可以是动态的，静态的，或使用随机子图像。下面的函数定义每个粒子的视觉基本形状：

  1. part_type_shape
  2. part_type_sprite

下图显示了14个内建粒子的类型，你可以选择：

一旦你已经选择粒子类型的精灵（无论是默认的形状或您自己的），你可以声明它的大小和比例。这些尺寸可被指定为一个范围，并全部的精灵按照选定的范围来创建。比例是“基础”的值，是在任何变化前的x和y轴的比例，下列函数是用来控制这些粒子的值：

  1. part_type_size
  2. part_type_scale

粒子也可以有一个方向和运动。正如大多数的粒子的功能，速度，方向和方向报都被解析为一个随机的值，该值是从一个给定的范围内选择，并且可以使用以下函数设置：

  1. part_type_speed
  2. part_type_direction
  3. part_type_gravity
  4. part_type_orientation

一旦你的风格和你的粒子运动的整理出来，然后你可以设置它们的颜色和alpha值，这两者都可以被设置为一个绝对值，在整个生命周期的粒子，或作为值从规模一个其他逐渐。请注意这里面还有一个特殊的功能，使您能够设置混合粒子及其周围，这样就可以正常或添加，将做出一些梦幻效果的功能，这些函数包括颜色，透明度和混合：

  1. part_type_color_mix
  2. part_type_color_rgb
  3. part_type_color_hsv
  4. part_type_color1
  5. part_type_color2
  6. part_type_color3
  7. part_type_alpha1
  8. part_type_alpha2
  9. part_type_alpha3
  10. part_type_blend

粒子存在屏幕上的一个有限的时间内，这被称为他们的“生命周期”。粒子寿命是测量步（像以前一样），这是表示为一个范围值的粒子将被赋予一个特定的生命周 期。也有一些有趣的功能，可用于使粒子产生其他粒子，每一个步骤或当他们“死”（即：从屏幕上消失）时，但这些功能必须非常小心，因为他们可以快速创建数 以千计的粒子并造成游戏系统崩溃！设置寿命和步骤或死亡发射器的函数如下：

  1. part_type_life
  2. part_type_step
  3. part_type_death

值得一提的是，创建粒子通过代码往往是一个费力和耗时的过程，由于大量的参数，可以改变和调整。由于这个原因，建议您四处搜寻，并保持某种类型的粒子设计器（或自己做一个！）的粒子产生的苦差事。
