Advertising and Analytics 广告与分析

这些功能为集成提供广告和分析服务

有许多不同的公司提供了增加广告到你的游戏的机制,还有其他让您可以将某种形式的信息分析整合起来(这样你可以看看你的游戏是在给定的市场扩张的区域)。GMS有一些函数包括了这些标准，使添加这些服务是一件简单的事情。

NOTE: 这些功能仅 Professional Version 专业版可用.

In Game Advertising 游戏中的广告

一旦您 Global Game Settings - Advertising Tab 中设置了提供广告, 您将需要把它们写进你的游戏，使用以下简单的函数来控制如何使用和显示的广告:

  1. ads_enable
  2. ads_disable
  3. ads_move
  4. ads_get_display_width
  5. ads_get_display_height

我们也支持基本的间隙广告，见如下函数:

  1. ads_interstitial_available
  2. ads_interstitial_display

除了这些基本的广告,有些广告提供商(如Supersonic 和AdColony)提供更多的互动间隙广告可以用于生成用户的虚拟货币。然而对于这些广告正确运行,您将需要建立自己的数据库记录为每个玩家的货币,这 也意味着你应该创建自己的登录系统让玩家id,或者有一些机制来识别每个安装你的游戏到数据库。

当你激活其中一个广告它可以是一个“offerwall”(实质上是一个页面来促进几个不同的广告活动)或一个视频,当玩家浏览广告或点击价格 单,广告公司在服务器上调用你设置了的回调而奖励人民币。注意,它可能是15分钟后玩家观看了电影(或点击价格单),或者它可能是瞬时所以你必须有后台数 据库的建立和一天24小时活动的。

NOTE : 为您的玩家检索货币，你需要使用 http_get 和 http_post_string 函数.

以下函数为公司广告而存在:

  1. ads_setup
  2. ads_engagement_available
  3. ads_engagement_launch
  4. ads_engagement_active

Testing Ads 测试广告

在您最终使用之前某些提供商提供的在测试在您的设备上,所以GMS中的选择开关测试 Global Game Settings - Advertising Tab. 这个选项会影响你的应用程序在目标平台上的不同选择和提供者。

Android

在Android设备上测试时被激活时,所有的广告将会显示一个蓝色背景,允许你看到什么位置和面积被用于广告以及观察测试广告符合指定的广告空间。并不是所有的广告提供商允许广告测试，所以下面是一个简短的总结那些做和如何设置它们：

  • AdMob - 你必须首先运行游戏一次AdMob广告(不是测试!),确保你的key0广告关键是正确的和使用 ads_enable 函数。如果你再看行控制台输出,底部附近您应该看到下面的线: 设备id号码,那么应该复制到适当的空间在 Advertising tab - global Game Settings,您现在应该国旗“test Ads”复选框切换广告测试。当你运行你的游戏再一次,这将是确认工作控制台输出显示消息“Admob using test adverts”。

  • Ad Colony - 你可以看到他们的测试视频和报价墙，通过配置测试选项通过你的页面访问他们的网站 http://www.adcolony.com/

  • Millennial Media - 你应该设置key0 app id为“28911”,国籍“Test Ads”复选框开始测试广告在你的设备上。这也将提高数量的事件日志登记使调试更加简单。

iOS

iOS 广告测试是限制的，只有少数公司目前和下面列出的这些连同介绍如何设置:

  • AdMob - 不像在Android设备,Admob的iOS只需要激活的广告选项卡,勾选“Test Ads”复选框,然后它会工作,发送测试广告和忽视点击

  • iAds iAds只会使用测试广告如果您正在运行一个签署了分布构建和这是无论设置在GMS因为它是自动完成iAds。

  • Millennial Media – 与Android,事件日志级别增加到调试水平,而是有一个通用的测试应用程序id,您必须注册您的设备id在 mmedia.com 网站上接受测试广告。

HTML5

目前支持HTML5测试广告必须通过控制面板配置你的开发者账户与所选择的公司,与“Test Ads”选项在全局游戏设置此刻什么也不做。它只是提供使未来的广告合作伙伴集成更加容易

Windows Phone

在Windows phone上测试广告,你仍然需要输入不同的广告单元的id,您希望有(无论是与真实id从微软pubCenter或“假”值来模拟 SDK将使用一个真正的广告)到不同的部分并选择你想要显示的广告类型。然后标记“Test Ads”选项在全局游戏设定能看到他们。注意,模拟器将始终表现出测试广告,不管你有什么设置。

Pocket Change 零用钱

零花钱是一种通用货币,允许您忠诚用户令牌的礼物零花钱玩你的游戏,这些令牌旅行与用户之间的游戏。然后用户可以交换他们对现实世界的礼物,例如折扣卡和专业的产品意味着这是一个很好的选择,当你正在考虑用户保留。口袋改变集成api GMS 非常简单易用,但会要求你首先 signed up for the Pocket Change 服务器.

当你已经做了,你可以使用以下功能在你的游戏(别忘了设置游戏是否正确 Global Game Settings - Pocket Change Tab):

  1. pocketchange_display_reward
  2. pocketchange_display_shop

Analytics 分析

你的游戏可以被不同的分析服务提供商 (主要 Flurry 和 Google Analytics) 可以设置在 Global Game Settings - Analytics Tab. 一般而言,分析是自动的,每个供应商跟踪产品和使信息可以通过他们自己的web页面,但是有两个额外的功能GMS提供的允许你发送文本事件分析提供者,然后您可以使用跟踪特殊功能在你的游戏:

  1. analytics_event
  2. analytics_event_ext
