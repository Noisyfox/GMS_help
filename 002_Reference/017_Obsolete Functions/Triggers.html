Trigger Events 触发器事件


触发事件是一种特殊的事件，将“触发”时，你在代码中指定的要求已得到满足。

为什么淘汰这些

触发器，实现由以前版本的游戏制作，在高效时，大多数人很少使用的，似乎造成游戏的逻辑和跟踪困难得多，比复杂的游戏。对于那些使用它们，可以很容易地实现相同的功能，通过创建一个脚本，把它在开始一步事件，step事件或事件的对象的结束步。

正如你可能已经使用触发器的游戏制作与以前版本的旧文件，你可以找到一个概要的触发事件是什么以及它是如何作为参考使用下面的：



触发器事件

要定义触发事件，你选择从“资源”菜单中定义触发器的主要游戏制作窗口的顶部，或者你可以选择在对象中添加一个触发器事件，然后选择“添加/修改触发器”。无论哪种方法，将打开下面的表单，您定义触发短代码功能。这可能是书面的形式如下图所示，包含一个例子。

在你的左边看到一个列表的所有触发器，迄今已定义的（显然是空的，如果你还没有创建任何！）。下面有一些按钮具有以下功能：

  • Add. 使用此按钮来添加一个新的触发事件的列表。形式将改变，以显示新的空触发准备添加必要的代码和名称。

  • Delete. 删除当前选定的触发事件的列表中选择。要小心，从来没有删除触发事件的对象中使用！这将导致不可预料的结果，肯定会引起你的问题在您的项目中。

  • Load. 触发器从文件加载的集合。这样的文件扩展名必须为.gmtrg。的触发来自该文件的左侧上的触发器的形式添加到列表中，然后，可以使用或修改为正常。

  • Save. 收集保存到一个文件中的触发器。必须在该文件的扩展名为.gmtrg。在这种方式中，您可以创建自己的库的触发器，然后可以加载到使用任何进一步的项目，并因此节省您的时间。

  • Clear. 彻底清除列表中的所有触发器。要小心，只使用对象时，没有使用任何触发事件！删除清除的对象正在使用的触发事件时，可以有不必要的副作用，可能会导致严重的问题，当运行您的项目。

一旦你完成，你必须按“确定”按钮，保存所做的更改，可以使用触发事件在对象。如果你不想保存更改，只需在十字架上，通过点击右上角关闭，并表明您不想保存更改，会弹出对话方块中的。

定义一个触发器

一旦你已经创建了一个触发器，您必须填写在其属性，一般开始给它起名字。此名称可以是任何你想要的，只要你使用的字符（见进一步下降的信息，触发命 名为一常数），但应该是描述，让你知道它的瞬间是什么，当你选择它使用在一个对象。此名称将出现在菜单可用的触发器，在事件列表中的对象。接下来你必须做 的是指定的条件，将“触发”触发事件！这是一块的GML代码到窗口上写的权利运行时，它返回一个true或false值。在这样一个触发器将触发时的返回 值是真实的。

使用范例

假设你要创建的事件发生，当玩家按下控制键和A键。对于这一点，你可以写以下条件脚本：

{
return keyboard_check(ord('A')) && keyboard_check(vk_control);
}


正如你可以看到的这种情况下，脚本将返回false，除非控制键和A键一起按下。

实际上，你可以把一些相当复杂的，完整的脚本为条件，只要在那里的某个地方，你有一个return语句的计算结果为true或false。例如，有一个事件发生，只有当实例的数量（在这种情况“obj_ball”）是在3和5之间，你可以使用下面的条件脚本：

{
var n;
n = instance_number(obj_ball);
return (n >= 3) && (n <= 5);
}


您可能会发现，有时你需要的代码来定义条件是太长的代码窗口内触发形式，所以你可能要在实际的代码编辑器中进行编辑，而不是。为此按下按钮使用代码编辑器中打开该窗口，并保持正常的工作，在左上角将更改保存到触发点击上打钩。

您可以创建许多其他类型的触发事件。例如，事件，请检查您是否靠近敌人，事件，请检查您是否收集到足够的钱或具有较高的分数，事件生成一个对象在某一个地方...可能性是无穷无尽的，这使得使用触发一个非常方便的方式来组织你的游戏，再利用脚本在其他项目中是有帮助的。

您所指定的条件脚本后，你应该指定要测试的条件。有三个选项，所有相关的步骤事件：

  • 你可以在开始步之前测试条件（就在开始步事件之前）
  • 你可以在步中时测试条件（就在步事件之前）
  • 或者你可以在步结束时测试条件（就在结束步事件前）

大多数情况下，默认的中间位置是好的，但在某些情况下，你需要的时刻，特别是如果触发的结果将影响未来的一步，或依赖于以前已经完成的东西。

命名一个常数

最后，你甚至可以创建一个常数，从代码中调用指定的触发事件。要做到这一点，你必须给它一个名称不能包含空格，也不是未经授权的字符，而且很容易记 住。如果你看看上面的图片，你可以看到，我们已经使用了“tr_Near_Wall”，触发定义常量。在此之前的命名规则，是描述性的，易于理解，将允许 从任何其他事件中的代码被称为触发。应该指出的是，在代码中使用触发器不计算为true或false，但它仍然可以被用来设置变量和其他的东西。代码使用 的样本将是：

event_perform(ev_trigger, tr_Near_Wall);


使用触发器

一旦你定义了你的触发器定义对象时，可以使用它们。要做到这一点，你就点击“添加事件，然后选择”触发事件“。

一个弹出会给你一个你已经创建或加载，并从那里，你可以选择你想使用的触发器列表。然后，它将被添加到事件列表为正常。
