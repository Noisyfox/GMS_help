Obsolete Display Functions 淘汰显示函数

这些函数处理游戏显示方面。

Function List :

FunctionDescription
screen_refresh使用当前房间图像刷新显示（而不是使用绘图事件）。
screen_wait_vsync等待至下一次的显示器垂直同步。
set_automatic_draw指定是否自动重绘房间（设为 true 则为真，默认。或者 false 为否）。
set_synchronization指定是否根据屏幕刷新率同步绘制屏幕。
display_set_size设定显示的宽度和高度，单位像素。返回是否成功。（记住只有正确的组合才被允许）。
display_set_colordepth设置色深。通常只有16位和32为两种允许值。返回是否成功。
display_set_frequency设置刷新率。通常只有很少的几个允许值（ 比如60、75、85等等）。返回是否成功。
display_set_all一次性设置。 -1 代表不更改。返回是否成功。
display_test_all测试返回给定的设置是否允许。 -1 代表不更改。
window_set_region_size设置窗口中的绘图区域宽度和高度。 adaptwindow 参数指定窗口尺寸是否一定要适应绘图区域大小。窗口尺寸在你使用固定缩放 比例时将总是适应。（请看函数 window_set_region_scale() 相关内容。）
window_set_region_scale如果窗口比当前房间大，通常房间就会显 示在窗口的中心位置。这里你可以指定房间自动缩放以适应窗口大小。设为 1 代表不进行 缩放。如果你使用 0 作为参数则代表伸展填充全部窗口区域。如果你设置为一个负值则代 表在窗口内维持高宽比的最大尺寸（通常也是你希望的模式）。adaptwindow 指定窗口尺寸 是否根据房间尺寸进行适应调整。只在缩放值为正值时起作用。
window_get_region_scale返回绘图区域的缩放值。
window_get_region_width返回当前绘图区域的宽度。
window_get_region_height返回当前绘图区域的高度。
window_set_visible设置游戏窗口是否可见。当然通常状况下你会希望游戏窗口 一直可见。处于不可见状态的窗口不会对任何键盘事件产生反应。
window_get_visible返回游戏窗口是否可见。
window_set_showborder设置是否显示窗口的边框（全屏模式下不会显示边框）。
window_get_showborder返回窗口是否显示边框。
window_set_showicons设置窗口图标（最小化，最大化，关闭）是否显示（全屏模 式下不会显示）。
window_get_showicons返回窗口是否显示边框图标。
window_set_stayontop设置窗口是否一直处于最前。
window_get_stayontop返回窗口是否处于最前。
window_set_sizeable设置窗口是否可以被用户调整。（玩家只能在边框被显示的非全屏模式下调整窗口大小）。
window_get_sizeable返回窗口是否可以被玩家调整大小。
window_default将窗口设置为默认尺寸和位置（居中）。
window_view_mouse_set在编号为 id 的视野内设置鼠标坐标为(x,y)。
window_views_mouse_set设置当前可见的第一个视野内的鼠标坐标为(x,y)。 

为什么淘汰这些

这些函数在改为GMS的屏幕绘制的方式中很落后，事实上这些函数在电脑和设备中已经被遗忘很长时间了，或者已经不在再工作与非windows设备，它们通常不是必要的。
