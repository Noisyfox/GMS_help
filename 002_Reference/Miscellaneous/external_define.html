external_define

Defines an external function.

Syntax :

external_define(dll, name, calltype, restype, argnumb, argtype[0], argtype[1], ...) ;


ArgumentDescription
dll The name of the dll file (string)
name The name of the function (sting)
calltype The calling convention used
restype The type of the result to expect
argnumb The number of arguments (0 - 15)
argtype[0 ... 15] The different types of arguments being used


Returns : N/A


Description

This function can be used to define an external function call to a specific dll (dynamic link library). You supply the name (and path) of the dll, then the name of the function that you wish to define. Next you need to define the calling convention to be used (see the constants list below) as well as the type of result to be expected (also a constant, as listed below). Finally you must give the number arguments that the function can take (from o to 15) and for each of the arguments you must specify it's type too. Please note that for functions with 4 or more arguments, all of them must be of type ty_real.

ConstantDescription
dll_cdeclThis is the default C, C++ call
dll_stdcallThis is the standard WinAPI call
ty_realA real number argument
ty_stringa null-terminated string argument


Example :

global.mmm = external_define('MyDLL.DLL', 'MyMin', dll_cdecl, ty_real, 2, ty_real, ty_real);

The above example code will define an external function call with two arguments and store the id in a global variable.
