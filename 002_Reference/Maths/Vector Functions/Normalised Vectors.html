Normalised Vectors 规范化矢量

本章节介绍什么是规范化矢量及如何使用。

通常情况下，在许多情况下可以使用矢量，但有时你想限制它们的值（如在处理角度），这就是为什么我们要规范化矢量。这基本上是一个数学的技巧，被用于转换的矢量长度为n的到长度为1矢量分量，这意味着，被规范化到0和1之间的矢量。这些矢量也被称为单位矢量：

要计算的单位向量，我们必须先有原来的矢量分量，然后用他们得到的矢量长度。然后，我们将每个由该长度的矢量分量得到的规范化的向量，构成因子形成的所有坐标的平方的总和等于1，其中的规范化的矢量。具体方法如下：

首先，我们要获取矢量坐标和获得组成部分：

vx = (x2 - x1) = (7 - 1) = 6
vy = (y2 - y1) = (4 - 1) = 3


然后，我们使用这些值来计算的向量长度:

len = sqr(vx2 + vy2) = sqr(36 + 9) = sqr(45) = 6.708203932499369


现在，这给了我们一个确切的矢量长度，这样可以让使用规范化的两个矢量分量VX和VY：

vx = (vx/len) = (6 / 6.708203932499369) = 0.8944271909999159
vy = (vy/len) = (3 / 6.708203932499369) = 0.4472135954999579
a = 1


太好了！现在，我们已经标准化矢量的组件！但在GameMaker:Studio下实际使用的是？好吧，让我们给您一个实际的例子...

假设你有一个游戏，玩家攻击一个敌人，你需要知道在每一步中有打出多少沿x和y轴移动的子弹对象：

对于这一点，你可以使用玩家和敌人的坐标，得到的矢量分量和长度，然后你会规范化0和1的值，你最终会乘以速度，你的子弹运行的每一步。这些最后的两个值，然后存储和添加到起始x坐标和y坐标到每个步骤。听起来很复杂？这不是，看（值已四舍五入至小数点后一位为简单起见）：

px = 100;
py = 425;
ex = 356;
ey = 83;
bullet_speed = 5;

vx = (ex - px) = 256
vy = (ey - py) = -342

len = sqr(vx2 + vy2) = sqr(65536 + 116964) = 427.2

vx = vx / len = 0.6
vy = vy / len = 0.8

speed_x = vx * bullet_speed = 3
speed_y = vy * bullet_speed = 4


因此，要击中目标，我们需要增加3到子弹x坐标，增加4到子弹的y坐标在每步中。
