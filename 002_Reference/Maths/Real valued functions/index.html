Real Number Functions 实数函数

如下函数是处理实数的。

在GameMaker:Studio 中的实数被认为是浮点数，即是说，他们在不固定数字位的数字中有小数点，或之前或之后。例如，2是一个二进制的整数，但2.0是一个浮点实数。在处理跨平 台开发的精密计算时，在Windows PC上与移动设备上的计算精度是不相同的，整数和浮点数之间的区别是非常重要的。这意味着，在进行比较时你应该特别注意，例如：

if image_index == 1 {do something...}


现在可以做显示工作了，但由于浮点数学工作的方式，它可能意味着image_index，实际上等于1.00000000001，在这种情况下，则将永远 不会得出正确的结果和运行的代码。这些类型的错误调试是十分困难的，所以最好是提前使用其他手段的检查值，或使用适当的取舍或取入的函数（如下所列）为检 查而把数字转换到一个整数。例如，上面的代码可以被写为：

if floor(image_index) == 1 {do something...}


在GameMaker:Studio里我们也可以使用一个特殊的函数，设置精度值为了浮点数学。一个实数舍入到最近的浮点数时，精度（也被称为“机器精度”）形成的相对误差的上限，并可以使用如下特殊函数设置的精度值：

  1. math_set_epsilon


我们可以把GameMaker:Studio处理各类实数的函数划分成几类：

随机函数

这些函数都是用于随机数和值的处理：

  1. choose
  2. random
  3. random_range
  4. irandom
  5. irandom_range
  6. random_set_seed
  7. random_get_seed
  8. randomize

NOTE : 当使用随机函数时， GameMaker:Studio保持相同的随机种子在每次启动游戏。这使得调试更加容易保证您的随机函数总是返回相同的值，但是如果你不希望这种情况发生，你必须首先设置一个新的随机种子在游戏开始时，使用 randomize or random_set_seed。
