Drawing Sprites And Backgrounds 绘制精灵与背景

本章节包含了所有可用于绘制图形类原生资资源的函数。

即使你可以设置背景在房间编辑器中，你可以指定精灵到实例，通常这在游戏中不够灵活。例如，如果你有一个实例，画一个菜单，你可以在同一时间要显示的各 种按键精灵，或者如果你有一个滚动视差背景的游戏，你将需要能够通过代码设置某些类型的控制器实例。在这种情况下，你应该使用下面的函数，可以在房间中的任何位置绘制精灵和背景的原生资源的函数，也允许你操纵精灵的形状，大小，颜色和透明度达到特殊的效果。

精灵

您可以指定一个精灵到一个实例，默认情况下， GameMaker:Studio 通过精灵变量 sprite variables 绘制它的各种属性, 但是，如果你添加一些什么到绘制事件，你是“获取控制”，必须准确地告诉GameMaker:Studio要画什么。意味着必须使用下面的函数来定义绘制什么实例，以及房间里绘制的位置：

  1. draw_self
  2. draw_sprite
  3. draw_sprite_ext
  4. draw_sprite_general
  5. draw_sprite_part
  6. draw_sprite_part_ext
  7. draw_sprite_stretched
  8. draw_sprite_stretched_ext
  9. draw_sprite_pos
  10. draw_sprite_tiled
  11. draw_sprite_tiled_ext

背景

虽然房间编辑器背景选项卡 Room Editor Backgrounds Tab 中有许多的选项设置背景，通常使用的代码绘制会更有用。为此，GameMaker:Studio 具有以下的函数，允许您以许多不同的方法绘制背景一类的原生资源：

NOTE : 不要忘了一个实例的绘制事件添加任何代码，将关闭该实例精灵自动绘制，所以如果你需要绘制精灵和背景，你应该结合上述函数和下面的函数进行绘制。

  1. draw_background
  2. draw_background_ext
  3. draw_background_part
  4. draw_background_part_ext
  5. draw_background_stretched
  6. draw_background_stretched_ext
  7. draw_background_tiled
  8. draw_background_tiled_ext
  9. draw_background_general
